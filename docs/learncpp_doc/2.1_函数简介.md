# 2.1 — 函数简介

---

在上一章中，我们把函数定义为“一组按顺序执行的语句”。这固然没错，但这个定义**并没有说明函数为什么有用**。所以我们更新一下这个定义：

> **函数是一段可复用的、用于完成某个特定任务的语句序列。**

你已经知道：每个可执行程序都必须有一个名为 `main()` 的函数（程序运行时会从这里开始执行）。但当程序越写越长，把所有代码都堆在 `main()` 里会**越来越难维护**。函数提供了一种方式，让我们能把程序**拆成若干个小而独立的模块**，更方便组织、测试和使用。大多数程序都会用到很多函数。C++ 标准库也提供了大量现成的函数供你调用——但**自己写函数**同样是家常便饭。**你自己写的函数被称为“用户自定义函数（user-defined function）”**。

想象一个日常场景：你正在看书，突然想起有个电话要打。于是你**在书里夹一张书签**，去打电话，打完之后**回到书签处继续看**，正好从你离开时的那一页那一行接着往下读。

C++ 程序也是这么工作的（连术语都借用得差不多）：程序正在某个函数里按顺序执行语句时，如果**遇到一次函数调用（function call）**，CPU 会“**中断当前函数**”，转去执行被调用的函数。CPU 相当于在当前执行位置**放了一张书签**，去执行那个被调用的函数，执行完再回到书签处，从那里继续往下执行。

> **命名约定**
>
> **发起函数调用的函数称为 caller（调用方），被调用的函数称为 callee（被调用方）。** 函数调用有时也被称为 **invocation（调用）**，caller **invokes**（调用）callee。

## 一个用户自定义函数的例子

先来看看**定义一个用户自定义函数的最基本语法**。接下来的几节里，我们所有的用户自定义函数都长这个样子：

```cpp
returnType functionName() // 这是函数头（告诉编译器这个函数的存在）
{
    // 这是函数体（告诉编译器这个函数要做什么）
}
```

- 第一行**非正式地**被称为**函数头（function header）**，它告诉编译器：这个函数存在，它叫什么名字，以及一些其他信息（比如返回类型，后面会讲）。
- 在本节里，我们使用的 `returnType` 只有两种：`int`（用于 `main()`）和 `void`（用于其他函数）。目前不用纠结它们的含义，我们会在下一节 2.2 — 函数返回值（有返回值的函数）中详细讨论。
- 就像变量有名字一样，用户自定义函数也有名字。`functionName` 就是这个函数的名字（标识符）。
- **标识符后面的一对圆括号 `()`** 是在告诉编译器：**我们正在定义一个函数**。
- 大括号 `{}` 内部的语句被称为**函数体（function body）**，你想让函数做什么，就写在这里。

**调用函数**时，写函数名后跟一对圆括号即可，例如 `functionName()` 会调用名为 `functionName` 的函数。**按惯例，圆括号紧挨着函数名，中间不加空格。**

目前来说，**函数必须先定义，才能被调用**。我们会在 2.7 — 前向声明与定义 中讲解如何绕开这个限制。

下面是一个示例程序，演示了一个用户自定义函数的定义和调用：

```cpp
#include <iostream>

// 用户自定义函数 doPrint() 的定义
// 在本例中，doPrint() 是被调用的函数（callee）
void doPrint()
{
    std::cout << "In doPrint()\n";
}

// 用户自定义函数 main() 的定义
int main()
{
    std::cout << "Starting main()\n";
    doPrint();                        // 中断 main()，去调用 doPrint()；这里 main() 是调用方（caller）
    std::cout << "Ending main()\n";   // 这条语句会在 doPrint() 执行完之后执行

    return 0;
}
```

程序输出：

```
Starting main()
In doPrint()
Ending main()
```

程序从 `main()` 顶部开始执行，第一行打印 `Starting main()`。

`main()` 的第二行是对函数 `doPrint()` 的调用（末尾有 `()`，所以我们能看出这是函数调用）。

> **警告**
>
> **调用函数时不要忘了函数名后的圆括号 `()`。** 忘了圆括号可能会导致编译失败；就算能编译，函数也**根本不会被调用**。

由于发生了函数调用，`main()` 中语句的执行被**挂起**，控制流跳到被调用函数 `doPrint()` 的顶部。`doPrint()` 中的第一行（也是唯一一行）打印 `In doPrint()`。`doPrint()` 结束后，控制流**返回调用方 `main()`**，从函数调用**下一条语句**继续执行。于是接着执行 `main()` 中的下一条语句，打印 `Ending main()`。

## 函数可以被多次调用

函数一个非常有用的特点是：**它可以被多次调用**。下面这段程序演示了这一点：

```cpp
#include <iostream>

void doPrint()
{
    std::cout << "In doPrint()\n";
}

int main()
{
    std::cout << "Starting main()\n";
    doPrint(); // 第一次调用 doPrint()
    doPrint(); // 第二次调用 doPrint()
    std::cout << "Ending main()\n";

    return 0;
}
```

程序输出：

```
Starting main()
In doPrint()
In doPrint()
Ending main()
```

因为 `main()` 调用了 `doPrint()` 两次，`doPrint()` 就会执行两次，`In doPrint()` 也就相应地被打印两次。

## 函数可以调用函数（函数调用可以嵌套）

你已经看到 `main()` 可以调用其他函数（比如上面的 `doPrint()`）。**被 `main()` 调用的函数也可以再调用其他函数**（这些函数还可以继续再调……）。下面这段程序里，`main()` 调用了 `doA()`，`doA()` 又调用了 `doB()`：

```cpp
#include <iostream>

void doB()
{
    std::cout << "In doB()\n";
}

void doA()
{
    std::cout << "Starting doA()\n";

    doB();

    std::cout << "Ending doA()\n";
}

int main()
{
    std::cout << "Starting main()\n";

    doA();

    std::cout << "Ending main()\n";

    return 0;
}
```

程序输出：

```
Starting main()
Starting doA()
In doB()
Ending doA()
Ending main()
```

## C++ 不支持嵌套函数

**定义在另一个函数内部的函数**叫做**嵌套函数（nested function）**。和某些语言不同，**C++ 不允许嵌套函数**。下面这段代码是**非法的**：

```cpp
#include <iostream>

int main()
{
    void foo() // 非法：这个函数嵌套在了 main() 内部
    {
        std::cout << "foo!\n";
    }

    foo(); // 调用 foo()

    return 0;
}
```

正确的写法应该是：

```cpp
#include <iostream>

void foo() // 不再嵌套在 main() 里
{
    std::cout << "foo!\n";
}

int main()
{
    foo();

    return 0;
}
```

> **命名约定**
>
> **`foo`** 是一个**没有实际含义**的词，常被用作示例中函数或变量的**占位名字**（当名字并不重要时）。这类词被称为 **元语法变量（metasyntactic variable）**——不过日常里大家更习惯叫它 **“占位名（placeholder name）”**，因为“metasyntactic variable”这个词根本没人记得住。C++ 中常见的元语法变量还有 **“bar”、“baz”**，以及一些以 “oo” 结尾的三字母词，比如 **“goo”、“moo”、“boo”**。
>
> 对词源学感兴趣的读者可以读一读 **RFC 3092**，很有意思。
