# 2.2 — 函数的返回值（有返回值的函数）

> 作者：Alex　发布时间：2025 年 2 月 5 日

看下面这段程序：

```cpp
#include <iostream>

int main()
{
    // 从用户获取一个值
    std::cout << "Enter an integer: ";
    int num{};
    std::cin >> num;

    // 输出它的 2 倍
    std::cout << num << " doubled is: " << num * 2 << '\n';

    return 0;
}
```

从概念上看，这段程序其实由两部分组成：**先从用户那里获取一个值**，再**告诉用户这个值的 2 倍是多少**。

虽然这段程序很短，还用不着拆成多个函数。但如果我们**想拆呢**？“从用户那里获取一个整数”本身就是一个**边界清晰的任务**，非常适合封装成一个函数。

于是我们试着这样写：

```cpp
// 这段程序有问题
#include <iostream>

void getValueFromUser()
{
    std::cout << "Enter an integer: ";
    int input{};
    std::cin >> input;
}

int main()
{
    getValueFromUser(); // 请用户输入

    int num{}; // 问题来了：怎么把 getValueFromUser() 里的值传出来赋给 num？

    std::cout << num << " doubled is: " << num * 2 << '\n';

    return 0;
}
```

这个思路是好的，但**实际上根本不能工作**。

`getValueFromUser` 被调用后，会如预期地要求用户输入整数。但当 `getValueFromUser` 结束、控制返回 `main` 时，**用户输入的值就丢了**。`num` 从未被真正初始化过用户输入的值，所以程序会**永远输出 0**。

**我们缺少的是**：让 `getValueFromUser` 把用户输入的值**返回给 `main`**，让 `main` 拿到这份数据。

---

## 返回值（Return values）

写一个用户自定义函数时，你可以决定**它是否要向调用方返回一个值**。如果要返回，需要做两件事。

### 1）指明返回类型

**函数必须先声明自己返回什么类型的值**，也就是设置**返回类型**——写在函数名前面的那个类型。上面例子里：

- `getValueFromUser` 的返回类型是 `void`，表示**不向调用方返回任何值**；
- `main` 的返回类型是 `int`，表示**会向调用方返回一个 `int` 类型的值**。

注意：**返回类型只决定“返回值是什么类型”，并没有决定具体返回什么值。**

> **相关内容**
>
> 关于返回类型为 `void` 的函数，我们会在下一节 2.3 — void 函数（无返回值的函数） 中进一步讨论。

### 2）在函数体里用 `return` 语句返回具体的值

在**要返回值**的函数里，我们用 **`return` 语句**指定要返回给调用方的具体值。它由：**关键字 `return`** + **一个表达式**（有时称为 **返回表达式**）+ **分号** 组成。

`return` 语句执行时会做三件事：

1. **计算返回表达式**，得到一个值；
2. **把这个值拷贝回调用方**，这份拷贝就是这个函数的**返回值（return value）**；
3. **函数结束**，控制权返回调用方。

这种“把值拷贝一份返回给调用方”的过程叫做**按值返回（return by value）**。

> **命名约定**
>
> **返回表达式**是那段用来计算出结果的表达式；**返回值**是这个结果被**拷贝**给调用方的那一份。

**有返回值的函数**每次被调用时都会返回一个值。看下面这个简单例子：

```cpp
#include <iostream>

// int 是返回类型
// 返回类型是 int，意味着这个函数会向调用方返回一个 int 值（具体是什么值，这里没规定）
int returnFive()
{
    // return 语句给出要返回的值
    return 5; // 把 5 返回给调用方
}

int main()
{
    std::cout << returnFive() << '\n';     // 输出 5
    std::cout << returnFive() + 2 << '\n'; // 输出 7

    returnFive(); // 合法：返回了 5，但 main() 没用它，所以被忽略

    return 0;
}
```

运行结果：

```
5
7
```

执行从 `main` 顶部开始。第一条语句里，`returnFive()` 被调用并求值：返回表达式 `5` 求值得到 `5`，被返回给调用方后由 `std::cout` 打印。

第二条语句里，`returnFive()` 再次被调用，返回 `5`；然后 `5 + 2` 求值得到 `7`，也被打印出来。

第三条语句里，`returnFive` 再一次被调用，同样返回 `5`。但 `main` 并**没有用**这个返回值，所以它就被**丢弃**了，什么也不会发生。

注意：**除非调用方主动把返回值通过 `std::cout` 输出出来，否则返回值不会自己打印**。上面第三个例子里返回值没有被送去输出，所以屏幕上什么都不会多出来。

> **小贴士**
>
> 当被调用的函数返回一个值时，调用方可以决定：**在表达式或语句里使用它**（比如用它初始化变量、或送去 `std::cout`），**或者干脆忽略它**（什么都不做）。**如果调用方忽略了返回值，这个值就被丢弃了**。

## 修好我们最开始那段有问题的程序

现在我们可以修好开头那段程序了：

```cpp
#include <iostream>

int getValueFromUser() // 现在这个函数返回一个整数
{
    std::cout << "Enter an integer: ";
    int input{};
    std::cin >> input;

    return input; // 把用户输入的值返回给调用方
}

int main()
{
    int num { getValueFromUser() }; // 用 getValueFromUser() 的返回值来初始化 num

    std::cout << num << " doubled is: " << num * 2 << '\n';

    return 0;
}
```

程序执行时，`main` 的第一条语句会创建一个 `int` 变量 `num`；在初始化它的时候，程序看到有一个对 `getValueFromUser()` 的函数调用，就先去执行这个函数。`getValueFromUser` 会让用户输入一个值，然后**把这个值返回给调用方 `main()`**。这个返回值被用来**初始化 `num`**。之后 `num` 就可以在 `main` 中随意使用了。

> **小贴士**
>
> 如果你需要**多次使用**某个函数调用的返回值，那就**用它初始化一个变量**，然后重复使用这个变量。

你可以自己编译并多次运行这段程序，验证它确实能正常工作。

## 重新审视 `main()`

现在你已经具备理解 `main()` 实际工作方式的概念工具了。

程序运行时，**操作系统会“调用” `main()`**。控制流跳到 `main()` 顶部，`main()` 内的语句依次执行，最后 `main()` **返回一个整数值（通常是 0）**，程序结束。

在 C++ 中，`main()` 有两个**特殊要求**：

1. `main()` **必须返回 `int`**；
2. **不允许显式调用 `main()`**。

```cpp
void foo()
{
    main(); // 编译错误：不允许显式调用 main
}

void main() // 编译错误：main 的返回类型必须是 int
{
    foo();
}
```

> **关键洞察**
>
> C 语言允许显式调用 `main()`，所以某些 C++ 编译器出于兼容性也允许这么做。目前来说，你应该把 `main()` 定义在**源文件的最下方**、其他函数的后面，并且**避免显式调用它**。

> **进阶阅读**
>
> 一个常见的误解是：`main` 一定是第一个被执行的函数。
>
> 其实**全局变量会在 `main` 执行之前被初始化**。如果这些全局变量的初始化器调用了某个函数，那么这个函数就会**在 `main` 之前被执行**。全局变量将在 7.4 — 全局变量简介 中讨论。

## 状态码（Status codes）

你可能会好奇：为什么 `main()` 通常 `return 0;`？什么时候要返回别的值？

`main()` 的返回值有时被称为**状态码（status code）**（也偶尔叫**退出码 exit code**、或者更少见的**返回码 return code**）。状态码用来**告诉外界你的程序是否成功执行**。

**按惯例，状态码为 `0` 表示程序正常运行**（即按预期执行完毕）。

> **最佳实践**
>
> **程序正常运行时，`main` 应该返回 `0`。**

**非 0 的状态码**通常表示某种“失败”。虽然在大多数操作系统上这样用没问题，但严格来说这**并不完全可移植**。

> **进阶阅读**
>
> C++ 标准只定义了三种状态码的含义：`0`、`EXIT_SUCCESS`、`EXIT_FAILURE`。
>
> - `0` 和 `EXIT_SUCCESS`：程序成功执行；
> - `EXIT_FAILURE`：程序执行失败。
>
> `EXIT_SUCCESS` 和 `EXIT_FAILURE` 是定义在 `<cstdlib>` 中的预处理宏：
>
> ```cpp
> #include <cstdlib> // 为了使用 EXIT_SUCCESS 和 EXIT_FAILURE
>
> int main()
> {
>     return EXIT_SUCCESS;
> }
> ```
>
> 如果你追求最大可移植性，成功时只用 `0` 或 `EXIT_SUCCESS`，失败时用 `EXIT_FAILURE`。
>
> 关于预处理器和预处理宏，我们在 2.10 — 预处理器简介 中介绍。

> **顺便一提**
>
> 状态码会被回传给操作系统。操作系统通常会把这个状态码提供给“启动了当前程序的那个程序”。这样，一个启动其他程序的程序就可以粗略地判断被启动的程序是否成功运行了。

## 有返回值的函数如果不返回值，会导致未定义行为

**返回值不是 `void` 的函数就叫做“有返回值的函数（value-returning function）”。**

**有返回值的函数必须通过 `return` 语句返回一个对应类型的值**，否则就是**未定义行为**。

> **相关内容**
>
> 关于未定义行为，见 1.6 — 未初始化变量与未定义行为。

下面是一个会导致未定义行为的例子：

```cpp
#include <iostream>

int getValueFromUserUB() // 声称返回一个 int
{
    std::cout << "Enter an integer: ";
    int input{};
    std::cin >> input;

    // 注意：没有 return 语句
}

int main()
{
    int num { getValueFromUserUB() }; // 用返回值初始化 num

    std::cout << num << " doubled is: " << num * 2 << '\n';

    return 0;
}
```

现代编译器**应该**会对此发出警告，因为 `getValueFromUserUB` 声明返回 `int`，但**没有 `return` 语句**。运行这样的程序会产生**未定义行为**，因为“一个有返回值的函数**没有真正返回值**”。

大多数情况下，编译器能检测出你是否忘了写 `return`。但在某些复杂情况下，编译器**未必能完全判断**你的函数是否在所有路径上都返回了值，所以**不要依赖编译器的检测**。

> **最佳实践**
>
> 请**确保所有返回类型不是 `void` 的函数，在所有路径上都通过 `return` 返回一个值**。
> 忘了返回值 → **未定义行为**。

---

## `main` 函数是唯一的例外：不写 `return` 时会隐式返回 `0`

前面这条规则**唯一的例外**就是 `main()`。如果 `main()` 没有 `return` 语句，它会**隐式地返回 `0`**。

尽管如此，**最佳实践仍是在 `main` 中显式写出返回值**——一来表明你的意图，二来与其他函数保持一致（其它函数不写返回值会导致 UB）。

---

## 函数每次只能返回一个值

一个有返回值的函数，**每次调用只能向调用方返回一个值**。

要注意的是：`return` 语句里给出的值**不必是字面量**——它可以是**任何合法表达式**的结果，包括一个变量、甚至另一个返回值函数的调用结果。上面 `getValueFromUser()` 里我们返回的就是变量 `input`——它保存了用户输入的那个数字。

至于**如何绕开“只能返回一个值”的限制**，我们以后会讲很多方式。

---

## “返回值代表什么”，由函数作者来决定

函数返回值的**含义**，是由**写这个函数的人**决定的。有些函数用返回值作为**状态码**，表示是否成功或失败；有些函数返回一个**计算出来或选出来的值**；还有些函数根本不返回任何东西（下一节我们就会看到例子）。

由于返回值的含义千差万别，**最好用注释说明清楚你的函数返回值代表什么**。例如：

```cpp
// 函数功能：请用户输入一个值
// 返回值：用户从键盘输入的整数
int getValueFromUser()
{
    std::cout << "Enter an integer: ";
    int input{};
    std::cin >> input;

    return input; // 把用户输入的值返回给调用方
}
```

## 复用函数

下面这个例子很好地体现了**函数复用**的价值。看这段程序：

```cpp
#include <iostream>

int main()
{
    int x{};
    std::cout << "Enter an integer: ";
    std::cin >> x;

    int y{};
    std::cout << "Enter an integer: ";
    std::cin >> y;

    std::cout << x << " + " << y << " = " << x + y << '\n';

    return 0;
}
```

它能跑，但**有明显的重复代码**。事实上，这段程序违反了良好编程中的一条核心原则：**Don’t Repeat Yourself（不要重复自己），常缩写为 DRY**。

为什么重复代码不好？

- 如果我们想把提示 `"Enter an integer:"` 改成别的内容，就得**改两个地方**；
- 如果我们要初始化的不是 2 个变量，而是 **10 个** 呢？重复代码会让程序更长、更难懂，也更容易出错。

我们把上面写好的 `getValueFromUser` 用上：

```cpp
#include <iostream>

int getValueFromUser()
{
    std::cout << "Enter an integer: ";
    int input{};
    std::cin >> input;

    return input;
}

int main()
{
    int x{ getValueFromUser() }; // 第一次调用
    int y{ getValueFromUser() }; // 第二次调用

    std::cout << x << " + " << y << " = " << x + y << '\n';

    return 0;
}
```

运行示例：

```
Enter an integer: 5
Enter an integer: 7
5 + 7 = 12
```

这里我们调用了 `getValueFromUser` 两次，一次初始化 `x`，一次初始化 `y`。**避免了重复代码**，也**减少了出错概率**。一旦确认 `getValueFromUser` 是对的，我们就可以放心地反复调用它，想调几次调几次。

这正是**模块化编程（modular programming）** 的精髓：**写一个函数 → 测试它 → 确认它工作 → 之后就可以无数次地放心复用**（前提是不再修改它，一旦改动就要重新测试）。

> **最佳实践**
>
> **遵循 DRY 原则：Don’t Repeat Yourself（不要重复自己）。**
> 如果某件事要做不止一次，就想办法减少代码里的重复：
>
> - **变量**可以用来保存需要多次使用的计算结果，避免反复计算；
> - **函数**可以用来定义需要多次执行的语句序列；
> - **循环**（以后会讲）可以用来重复执行同一条语句。
>
> 和所有最佳实践一样，DRY 是**指导原则**而非**绝对铁律**。有读者 Yariv 曾指出：**当代码被拆得过于零碎时，DRY 反而会伤害可读性。**

> **顺便一提**
>
> DRY 的反面（略带调侃）叫 **WET**：**Write Everything Twice（什么都写两遍）**。

---

## 小结

- **返回值**让函数可以**把一个值传回给调用方**。
- **函数**让我们能**尽量减少程序中的冗余代码**。
