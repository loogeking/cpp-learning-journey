# 1.9 — 字面量与运算符简介

---

## 字面量（Literals）

看下面这两条语句：

```cpp
std::cout << "Hello world!";
int x { 5 };
```

`"Hello world!"` 和 `5` 是什么？——它们是 **字面量（literal）**。**字面量（又叫字面常量，literal constant）** 是**直接写在源代码里的一个固定值**。

字面量和变量都有**值**和**类型**。不同之处在于：**变量的值可以通过初始化和赋值来设置和修改**，而**字面量的值是固定的、不能被改变**。字面量 `5` 永远就是 `5`。所以字面量也被称为**常量**。

为了进一步说明字面量和变量的区别，看下面这段短程序：

```cpp
#include <iostream>

int main()
{
    std::cout << 5 << '\n'; // 打印一个字面量的值

    int x { 5 };
    std::cout << x << '\n'; // 打印一个变量的值
    return 0;
}
```

- 第 5 行，我们向控制台输出值 `5`。编译器编译这段代码时，会生成让 `std::cout` 输出 `5` 的代码。这个值 `5` 会被**编译进可执行文件里**，可以被直接使用。
- 第 7 行，我们创建了一个名为 `x` 的变量，并用值 `5` 初始化它。编译器会生成一段代码，把字面量 `5` **拷贝**到分配给 `x` 的那块内存里。
- 第 8 行打印 `x` 时，编译器生成的代码会让 `std::cout` **从 `x` 所在的内存位置取出值（`5`）**并打印出来。

所以两条输出语句做的事情是**一样的**（都打印 `5`）。但字面量情况下，`5` 是**直接被打印**出来的；变量情况下，`5` 是**先从内存里取出来**、再被打印的。

这也解释了**为什么字面量是常量、而变量可以被改**：**字面量的值被直接放进可执行文件里**，可执行文件一旦生成就不能再改；而**变量的值放在内存里**，程序运行时**内存中的值是可以被修改的**。

> **关键洞察**
>
> **字面量是直接写在源代码里的值。** 这些值通常会**直接出现在可执行代码中**（除非被编译器优化掉了）。
>
> **对象和变量代表的是存放值的内存位置**，需要时可以从中**取出**这些值。

## 运算符（Operators）

在数学里，**运算（operation）** 是一个涉及**零个或多个输入值**（称为 **操作数（operand）**）、并产生一个**新值**（称为**输出值**）的过程。**具体执行哪种运算**由一个称为**运算符（operator）** 的符号来指示。

比如小时候我们都学过 `2 + 3 = 5`：这里字面量 `2` 和 `3` 是操作数，符号 `+` 是运算符，它告诉我们对这两个操作数做加法，从而得到新值 `5`。

在 C++ 中，运算也和你想的一样。例如：

```cpp
#include <iostream>

int main()
{
    std::cout << 1 + 2 << '\n';

    return 0;
}
```

在这段程序中，字面量 `1` 和 `2` 是 `+` 运算符的操作数，产生输出值 `3`，然后被打印到控制台。**在 C++ 中，运算的输出值通常被称为“返回值（return value）”**。

你多半已经很熟悉数学里那些常见的运算符了：加（`+`）、减（`-`）、乘（`*`）、除（`/`）。在 C++ 中，赋值 `=` 也是一个运算符，插入 `<<`、提取 `>>`、以及相等判断 `==` 也都是运算符。**大多数运算符是符号**（比如 `+`、`==`），但也有一些运算符是**关键字**（比如 `new`、`delete`、`throw`）。

> **作者笔记**
>
> 出于一些等到我们更深入讨论运算符时才会讲清楚的原因，对于符号形式的运算符，惯例上会把它的符号**接在单词 `operator` 后面**来指代它。
>
> 例如：加号运算符会写作 `operator+`，提取运算符会写作 `operator>>`。

一个运算符**接受的操作数个数**叫做它的 **元数（arity）**。这个词几乎没人认识，所以别指望在闲聊里用它别人能听懂。C++ 中的运算符按元数分为 4 种：

- **一元运算符（unary）** 作用于**一个**操作数。例如 `operator-` 的一元形式：`-5` 中，`operator-` 接受操作数 `5`，将其取反变成 `-5`。
- **二元运算符（binary）** 作用于**两个**操作数（常称为 **左操作数** 和 **右操作数**，分别位于运算符的左右两边）。例如 `3 + 4` 中，`operator+` 用左边的 `3` 和右边的 `4` 相加，得到新值 `7`。插入运算符 `<<` 和提取运算符 `>>` 都是二元运算符——左边是 `std::cout` 或 `std::cin`，右边是要输出的值或要输入到的变量。
- **三元运算符（ternary）** 作用于**三个**操作数。C++ 中只有一个（**条件运算符**），后面再讲。
- **零元运算符（nullary）** 作用于**零个**操作数。C++ 中也只有一个（**throw 运算符**），后面再讲。

要注意：**同一个运算符，根据用法不同可能有多种含义**。例如 `operator-` 就有两种上下文：**一元形式**用于对一个数取反（把 `5` 变成 `-5`，反之亦然），**二元形式**用于做减法（如 `4 - 3`）。

## 运算符的链式使用

运算符可以**串起来**，让一个运算符的**输出**作为另一个运算符的**输入**。以 `2 * 3 + 4` 为例：

1. 乘法运算符先执行，用左操作数 `2` 和右操作数 `3` 得到返回值 `6`；
2. 这个 `6` 成为加法运算符的左操作数；
3. 加法执行，用左操作数 `6` 和右操作数 `4` 得到新值 `10`。

关于运算符的**执行顺序**，我们在专门讨论运算符的章节里会更深入地讲解。现在只需要知道：**算术运算符的执行顺序和普通数学中的顺序相同**——**先括号，再指数，然后乘除，最后加减**。这个顺序常被缩写为 **PEMDAS**，或者展开为一个便于记忆的英文口诀：**“Please Excuse My Dear Aunt Sally”**。

> **作者笔记**
>
> 在一些国家/地区，PEMDAS 也被写成 **PEDMAS、BEDMAS、BODMAS 或 BIDMAS**。

## 返回值与副作用（Side Effects）

C++ 中大多数运算符只是**利用操作数计算出一个返回值**。例如 `-5` 得到返回值 `-5`，`2 + 3` 得到返回值 `5`。**也有少数运算符不产生返回值**（例如 `delete` 和 `throw`），我们以后会讲。

有些运算符还带有**额外的行为**。**如果一个运算符（或函数）除了产生返回值之外还带有一些可观察到的效果，我们就说它有“副作用（side effect）”。** 例如：

- `x = 5` 的副作用是**把 5 赋给了变量 `x`**。哪怕这个运算符已经执行完，`x` 的新值仍然是可以被观察到的（比如再打印一下 `x`）。
- `std::cout << 5` 的副作用是**把 5 打印到了控制台上**。哪怕这条语句执行完了，我们也仍能看到 `5` 已经出现在了控制台上。

> **命名约定**
>
> 在日常语言里，“**side effect**” 通常指某件事发生时**附带的（往往是负面的、意料外的）结果**（比如吃药）。例如口服抗生素常见的“副作用”之一是拉肚子。因此在日常语境里，副作用往往是**我们想避免、或与主要目的无关**的东西。
>
> 但在 C++ 里，“副作用”是一个**中性术语**：**它指一个运算符或函数除了返回值之外产生的可观察效果**。
>
> 由于赋值会带来“对象值被改变”的可观察效果，所以它被视为一种副作用。**我们使用某些运算符（例如赋值运算符）时，主要就是为了它们的副作用（而不是它们的返回值）**。在这种情况下，副作用是**有益且可预测的**（返回值反倒常常是次要的）。

> **进阶阅读**
>
> 对那些**主要为了返回值**才调用的运算符（比如 `operator+`、`operator*`），返回值是什么通常一目了然（比如两个操作数的和、积）。
>
> 但对那些**主要为了副作用**才调用的运算符（比如 `operator=`、`operator<<`），它们的返回值是什么就未必显而易见了。例如你觉得 `x = 5` 的返回值是什么？
>
> 事实上，`operator=` 和 `operator<<`（用于向控制台输出时）**都会返回它们的左操作数**。因此：
>
> - `x = 5` 返回的是 `x`；
> - `std::cout << 5` 返回的是 `std::cout`。
>
> 这样做是为了让它们**能被链式使用**。
>
> 例如 `x = y = 5` 会被解释为 `x = (y = 5)`：先执行 `y = 5`，把 `5` 赋给 `y`，这个表达式又返回 `y`；然后再把 `y` 赋给 `x`。
>
> 又如 `std::cout << "Hello " << "world!"` 会被解释为 `(std::cout << "Hello ") << "world!"`：先把 `"Hello "` 打印到控制台，这一步返回 `std::cout`，然后再用它把 `"world!"` 打印出来。
>
> 关于运算符的求值顺序，我们会在 6.1 — 运算符优先级与结合性 中更深入地讨论。
