# 1.10 — 表达式简介

---

## 表达式（Expressions）

先看下面这一组语句，每一条都定义了一个变量并对它进行初始化：

```cpp
// five() 是一个返回值为 5 的函数
int five()
{
    return 5;
}

int main()
{
    int a{ 2 };             // 用字面量值 2 初始化变量 a
    int b{ 2 + 3 };         // 用计算得到的值 5 初始化变量 b
    int c{ (2 * 3) + 4 };   // 用计算得到的值 10 初始化变量 c
    int d{ b };             // 用另一个变量的值 5 初始化变量 d
    int e{ five() };        // 用函数返回值 5 初始化变量 e

    return 0;
}
```

注意，这些初始化器里用到了各种不同的东西：**字面量、变量、运算符、函数调用**。C++ 会把这些不同的东西**统一转换成一个单独的值**，然后再把这个值作为变量的初始值。

那么这些初始化器有什么共同点呢？答案是：**它们都用到了“表达式”**。

在一般编程中，**表达式（expression）** 就是一段**非空**的字面量、变量、运算符和函数调用的组合，**它会计算出一个值**。执行表达式的过程叫做 **求值（evaluation）**，求出来的那个值叫做 **表达式的结果（result）**（有时也叫**返回值**）。

> **进阶阅读**
>
> 在 C++ 中，表达式的结果可能是以下三种情况之一：
>
> - **一个值**（最常见）；
> - **一个对象或一个函数**。返回对象的表达式会在 12.2 — 值类别（左值与右值） 中讨论；
> - **什么都不是**。这种情况出现在**没有返回值**的函数调用中（会在 2.3 — void 函数（无返回值函数） 中讲到）——这类调用**只是为了它们的副作用**。
>
> 目前为了简单起见，我们**假设表达式都会求值得到一个值**。

对一个表达式求值时，会**依次对它内部的各个部分求值**，直到最终只剩下一个值。下面是各种表达式的例子，右边的注释是它们的求值结果：

```cpp
2               // 2 是字面量，求值结果就是 2
"Hello world!"  // "Hello world!" 是字面量，求值结果是文本 "Hello world!"
x               // x 是变量，求值结果是变量 x 当前保存的值
2 + 3           // operator+ 用操作数 2 和 3 求值得到 5
five()          // 求值结果是函数 five() 的返回值
```

可以看到：**字面量**的求值结果就是它自己；**变量**的求值结果是它当前保存的值；**运算符**（比如 `operator+`）会用它的操作数计算出另一个值。函数调用我们还没正式讲，但在表达式的上下文中，**函数调用的求值结果就是函数返回的那个值**。

> **进阶阅读**
>
> 带**副作用**的运算符构成的表达式稍微复杂一点：
>
> ```cpp
> x = 5           // 副作用：把 5 赋给 x；求值结果是 x
> x = 2 + 3       // 副作用：把 5 赋给 x；求值结果是 x
> std::cout << x  // 副作用：把 x 的值打印到控制台；求值结果是 std::cout
> ```

> **关键洞察**
>
> **在 C++ 中，凡是需要一个“单独的值”的地方，你都可以放一个能产生值的表达式**——这个表达式会被求值成一个单独的值。

**表达式本身不以分号结尾**，也**不能单独编译**。例如，如果你想直接编译 `x = 5` 这个表达式，编译器会报错（大概率会抱怨少了个分号）。表达式**总是作为语句的一部分**参与求值。

例如下面这条语句：

```cpp
int x{ 2 + 3 }; // 2 + 3 是表达式，它没有分号——分号在整条语句的最后
```

如果把这条语句拆开来看，它的结构是这样的：

```
type identifier { expression };
```

- `type` 可以是任何合法类型（这里选的是 `int`）；
- `identifier` 可以是任何合法名字（这里选的是 `x`）；
- `expression` 可以是任何合法表达式（这里选的是 `2 + 3`，包含两个字面量和一个运算符）。

## 表达式语句（Expression statements）

某些表达式（比如 `x = 5`）主要是为了它们的**副作用**（这里是“把 5 赋给 `x`”）而使用的，而**不是为了它们的返回值**。

> **相关内容**
>
> 副作用相关内容参见 1.9 — 字面量与运算符简介。

但上面提到过：表达式**不能单独执行**——它必须**作为语句的一部分**存在。幸运的是，把任何一个表达式转换成一条等价的语句非常容易：**在表达式后面加一个分号**，就变成了一条 **表达式语句（expression statement）**。表达式语句被执行时，其中的表达式就会被求值。

所以我们可以把任意表达式（比如 `x = 5`）变成一个能通过编译的表达式语句（`x = 5;`）。

**当一个表达式作为表达式语句使用时，它产生的结果会被丢弃**（因为没被用到）。例如，`x = 5` 被求值时，`operator=` 的返回值就被丢弃了。这没什么问题，因为我们本来就只是想把 5 赋给 `x`。

---

## 无用的表达式语句

我们也可以写出**能编译、但毫无作用**的表达式语句。例如 `(2 * 3;)` 这样的表达式语句，其中的表达式求值结果是 `6`，然后被直接丢弃。虽然语法合法，但这种表达式语句是**没用的**。有些编译器（如 gcc 和 Clang）如果检测到这种“无用的表达式语句”，会给出警告。

## 子表达式、完整表达式与复合表达式

我们偶尔需要谈论某些特定种类的表达式。为此定义几个相关的术语。

看下面这几个表达式：

```cpp
2               // 2 是字面量，求值为 2
2 + 3           // 使用 operator+ 求值为 5
x = 4 + 5       // 4 + 5 求值为 9，再赋给变量 x
```

简单地说：

- **子表达式（subexpression）** 是**被当作操作数使用的表达式**。例如 `x = 4 + 5` 的子表达式是 `x` 和 `4 + 5`；而 `4 + 5` 的子表达式是 `4` 和 `5`。
- **完整表达式（full expression）** 是**不作为其他表达式子表达式的表达式**。上面三个（`2`、`2 + 3`、`x = 4 + 5`）**都是完整表达式**。
- 在口语层面上，**复合表达式（compound expression）** 指**包含两个或更多运算符使用**的表达式。`x = 4 + 5` 就是复合表达式（用到了 `operator=` 和 `operator+`）；而 `2` 和 `2 + 3` **不是**复合表达式。
