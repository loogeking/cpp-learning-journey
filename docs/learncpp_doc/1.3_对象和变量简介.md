# 1.3 — 对象与变量简介

---

## 数据与值

在 [1.1 — 语句与程序结构](1.1_语句与程序结构.md) 中，你已经学到：**程序中的绝大多数指令都是语句，而函数是一组按顺序执行的语句集合**。函数中的语句会执行一些操作，（希望能）产出这个程序被设计来完成的结果。

那么，程序究竟是**怎么产生结果**的呢？答案是：**通过操作（读取、修改和写入）数据**。在计算领域中，**数据（data）** 指的是任何可以被计算机移动、处理或存储的信息。

> **关键洞察**
>
> **程序是一组用来操作数据、以产生期望结果的指令的集合。**

从技术上讲，计算机程序（无论是源代码还是编译后的可执行文件）本身也是数据，因为它们同样可以被移动、处理和存储。不过在讨论程序时，我们通常用 “**代码（code）**” 来指程序本身，用 “**数据（data）**” 来指程序为产生结果而处理的信息。

一个程序可以通过多种方式获取要处理的数据：从文件或数据库读取、通过网络接收、由用户在键盘上输入，或者由程序员**直接把数据写在源代码里**。在前面提到的 “Hello world” 程序中，文本 `"Hello world!"` 就是被直接写进源代码里的，作为程序要使用的数据。程序随后把这份数据发送给显示器进行显示，从而完成对它的“处理”。

在编程中，一份单独的数据叫做一个**值（value）**（有时也叫**数据值 / data value**）。常见的值有：

- **数字**（例如 `5` 或 `-6.7`）
- **字符**：写在**单引号**之间（例如 `'H'` 或 `'$'`）。**只能包含一个符号**。
- **文本**：写在**双引号**之间（例如 `"Hello"` 或 `"H"`）。文本可以包含 0 个或多个字符。

> **关键洞察**
>
> - 写在**单引号**里的值会被编译器解释为**字符值**；
> - 写在**双引号**里的值会被编译器解释为**文本值**；
> - **数字值不加引号。**

直接写在源代码中的值被称为**字面量（literal）**。我们会在 5.2 — 字面量 中详细讨论它们。

对值最常见的操作之一，就是**把它们打印到屏幕上**。例如：

```cpp
#include <iostream> // 引入 std::cout

int main()
{
    std::cout << 5;       // 打印字面量数字 5
    std::cout << -6.7;    // 打印字面量数字 -6.7
    std::cout << 'H';     // 打印字面量字符 H
    std::cout << "Hello"; // 打印字面量文本 Hello

    return 0;
}
```

如果字符值或文本值**没有正确加引号**，编译器就会尝试把它当作 C++ 代码来解析，这几乎必然会导致编译错误。

字面量是给程序提供数据最简单的方式，但它有一些不足：**字面量是只读的**，它们的值不能被修改。因此，如果我们希望**在内存中存储数据**，就需要用别的方式来做到这一点。

---

## 随机存取内存（RAM）

计算机的主内存被称为**随机存取内存（Random Access Memory，简称 RAM）**。当我们运行一个程序时，操作系统会把它加载到 RAM 中。所有被硬编码到程序里的数据（例如 `"Hello, world!"` 这样的文本），也是在这一步被加载进内存的。

操作系统还会额外分配一些 RAM 供程序在运行期间使用。这部分内存常见的用途包括：

- 存放用户输入的值；
- 存放从文件或网络读入的数据；
- 存放程序运行过程中计算得到的值（例如两个数的和），以便后面再用。

你可以把 RAM 想象成**一排编号的小盒子**，程序运行期间可以用它们来存放数据。

在一些早期的编程语言中（例如 Applesoft BASIC），你可以**直接访问这些盒子**（比如可以写一条语句：“去取 7532 号盒子里的值”）。

---

## 对象与变量

在 C++ 中，**不鼓励直接访问内存**。作为替代，我们通过**对象（object）**间接访问内存。**对象代表一块存储区域**（通常是 RAM 或 CPU 寄存器），可以用来存放一个值。对象还带有一些附加的属性（后面会讲到）。

编译器和操作系统是如何为对象分配内存的，超出了本课范围。这里的关键点是：**我们不用再说“去取 7532 号盒子里的值”**，而是说 **“去取这个对象里保存的值”**，让编译器去处理具体到哪里、以什么方式取值。这意味着我们可以专注于**用对象来存取值**，而不必操心它们到底放在内存的哪个位置。

尽管 C++ 中的对象也可以是**无名的（匿名对象）**，但更常见的做法是**用一个标识符（identifier）给对象命名**。**带有名字的对象叫做变量（variable）**。

> **关键洞察**
>
> **对象** 用于在内存中存储一个值；**变量** 是一个**带有名字（标识符）的对象**。

给对象起名字，是为了让我们能在程序后面再次引用它们。

> **命名约定**
>
> 在一般编程语境中，**object** 通常指内存中的一段无名对象、变量或函数。而在 **C++** 里，**object** 的定义更狭义，**不包括函数**。在本教程中，只要我们说 “object”，指的都是这个 C++ 中较狭义的定义。

## 变量定义（Variable definition）

要在程序里使用一个变量，我们必须先告诉编译器：“我要一个变量”。最常见的做法是使用一种特殊的声明语句，叫做**定义（definition）**（我们会在 2.7 — 前向声明与定义 中说明「声明」和「定义」的区别）。

> **关键洞察**
>
> **定义语句** 用来告诉编译器：**我们想在程序里使用一个变量。**

下面是定义一个名为 `x` 的变量的例子：

```cpp
int x; // 定义一个名为 x 的变量（类型为 int）
```

在**编译期**（程序被编译时），编译器读到这条语句后，就会记下：我们需要一个名字叫 `x` 的变量，它的数据类型是 `int`（关于数据类型稍后会讲）。从此以后（有一些我们以后会讲到的作用范围限制），只要我们在代码里再次使用标识符 `x`，编译器就知道我们指的是这个变量。

关于这个变量的其它所有细节，编译器都会帮我们处理，包括：

- 这个对象需要多少内存；
- 对象将被放在哪种存储中（例如 RAM 还是 CPU 寄存器）；
- 它相对其他对象的位置在哪；
- 它什么时候被创建、什么时候被销毁……

一个变量通过定义语句**在该定义语句所在的位置被“定义”**。目前来说，你的变量都应该定义在函数内部（比如 `main()` 里）。

下面是一个包含变量定义语句的完整程序：

```cpp
int main()
{
    int x; // 变量 x 的定义

    return 0;
}
```

## 变量的创建（Variable creation）

在**运行时**（程序被加载进内存并开始运行时），**每个对象都会被分配一个真实的存储位置**（例如 RAM 或某个 CPU 寄存器），用来存放值。这个**为对象保留存储空间的过程叫做“分配（allocation）”**。分配一旦完成，对象就算被**创建**了，可以被使用了。

举个例子，假设变量 `x` 被实例化到内存位置 140。那么每当程序使用变量 `x` 时，实际访问的就是 140 号位置中的值。

> **关键洞察**
>
> **只有当实际存储空间被保留下来给一个对象使用时，这个对象才算被“创建”。**

上面的示例程序运行时，执行会从 `main()` 顶部开始，然后为 `x` 分配内存，接着程序结束。

## 数据类型（Data types）

到目前为止我们说过：对象是**能存放数据值的存储区域**（数据究竟是如何存储的，是以后要讲的话题）。而**数据类型（data type，通常简称 type）** 决定了这个对象要存放**什么样的值**（例如数字、字母、文本等）。

在上面的例子中，我们的变量 `x` 被赋予的类型是 `int`，这意味着 `x` 会存放一个**整数值**。**整数（integer）** 就是那些**不带小数部分**的数字，例如 `4`、`27`、`0`、`-2`、`-12`。简言之，我们可以说 `x` 是一个**整型变量**。

在 C++ 中，一个对象的类型**必须在编译期就已确定**，并且**不能在不重新编译程序的前提下被改变**。这意味着一个整型变量**只能**保存整数值。如果你想保存别的类型的值，就得使用**不同的类型**。

> **关键洞察**
>
> 一个对象的**数据类型必须在编译期就已知**（这样编译器才知道要为它分配多少内存）。

整数只是 C++ 内建支持的众多类型之一。举个例子，下面是使用 `double` 类型定义变量的例子：

```cpp
double width; // 定义一个名为 width、类型为 double 的变量
```

C++ 还允许你**创建自己的自定义类型**。后面我们会经常这么做，这也是 C++ 的强大之处之一。

在这几个入门章节中，我们主要使用整型变量，因为它们在概念上比较简单。但很快我们就会探索 C++ 提供的许多其他类型（包括 `double`）。

## 一次定义多个变量

C++ 允许在**同一条语句中**定义**多个类型相同**的变量，各变量名之间用**逗号**分隔。下面这段代码：

```cpp
int a;
int b;
```

和下面这段基本等价：

```cpp
int a, b;
```

使用这种写法时，初学者常犯 3 类错误（都不严重，编译器会帮你指出并要求修正）。

**第一种错误：**在连续定义变量时，给每一个变量都写上类型：

```cpp
int a, int b; // 错误（编译报错）

int a, b;     // 正确
```

**第二种错误：**试图在同一条语句里定义**不同类型**的变量，这是不允许的。不同类型的变量必须放在**不同的语句**里定义：

```cpp
int a, double b; // 错误（编译报错）

int a; double b; // 正确（但不推荐）

// 正确且推荐（更易读）
int a;
double b;
```

第三种问题会在下一节讲变量的**初始化**时讨论。

> **最佳实践**
>
> 虽然语法允许你在一条语句里定义多个同类型变量，但**尽量不要这么做**。应当**为每个变量单独用一条语句定义、写在单独的一行**（并配上单行注释说明它的用途）。

---

## 小结

在 C++ 中，我们**通过对象来访问内存**。**带有名字的对象叫做变量**。每个变量都有：

- 一个**标识符（identifier）**；
- 一个**类型（type）**；
- 一个**值（value）**；
- 以及其它一些与本节无关的属性。

变量的**类型**用于决定：内存中的值应该被如何解释。

**变量真正的创建发生在运行时——即为它们分配存储空间的那一刻。**
