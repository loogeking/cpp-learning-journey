# 1.7 — 关键字与标识符命名

## 关键字（Keywords）

C++ 为自己保留了一组词汇（截至 C++23，共 **92 个**）供其自身使用。这些词被称为**关键字（keyword）**，也叫**保留字（reserved word）**。每一个关键字在 C++ 语言中都有其**特定含义**。

以下是 C++（截至 C++23）所有关键字的列表：


标注了 **(C++20)** 的关键字是 C++20 新增的。如果你的编译器不支持 C++20（或者虽然支持但默认没开启），这些关键字可能还不能使用。

C++ 还定义了一些**特殊标识符（special identifiers）**：`override`、`final`、`import`、`module`。它们**只有在特定上下文中**才具有特殊含义，其他情况下并**不属于保留字**。

你已经接触过其中一部分关键字了，比如 `int` 和 `return`。这些关键字、特殊标识符加上一组运算符，**共同构成了整个 C++ 语言**（不包括预处理器指令）。因为关键字和特殊标识符具有特殊含义，你的 IDE 通常会**用不同颜色**把它们高亮出来，与普通标识符加以区分。

学完整套教程之后，你将会理解上面这些词几乎每一个的用途！

## 标识符的命名规则

再回顾一下：**变量、函数、类型或其他项目的名字被称为标识符（identifier）**。C++ 允许你相当灵活地命名标识符，但**必须遵守下面几条规则**：

1. **标识符不能是关键字**（关键字是保留的）。
2. **标识符只能由字母（大小写皆可）、数字和下划线 `_` 组成**。这意味着名字里**不能有其它符号（除了下划线），也不能有空白字符（空格或制表符）**。
3. **标识符必须以字母（大小写皆可）或下划线开头**，**不能以数字开头**。
4. **C++ 区分大小写**，因此 `nvalue`、`nValue` 和 `NVALUE` 是三个**不同的**标识符。

---

## 标识符命名的最佳实践

规则告诉了你**能怎么命名**，接下来说说你**应该怎么命名**变量（或函数）。

### 1）变量名通常以小写字母开头

C++ 中的**惯例**是：**变量名以小写字母开头**。如果变量名是**单个词或缩写**，那就**全部小写**。

```cpp
int value; // 合乎惯例

int Value; // 不合惯例（应以小写字母开头）
int VALUE; // 不合惯例（应以小写字母开头，且不该全大写）
int VaLuE; // 不合惯例（快去看看心理医生）;)
```

**函数名**大多数情况下也是以小写字母开头的（这一点存在一些分歧）。我们会遵循这个惯例，因为所有程序都必须有的 `main` 函数就是小写开头的，C++ 标准库中的所有函数也都是小写开头的。

**以大写字母开头**的标识符名，通常是留给**用户自定义类型**用的（例如 `struct`、`class` 和枚举等，我们以后会讲到）。

### 2）多单词名字的两种常见风格

如果变量或函数名包含**多个单词**，通常有两种命名风格：

- 单词之间用**下划线**分隔（有时叫 **snake_case**）；
- **驼峰式**（叫 **camelCase**，因为大写字母像骆驼的驼峰一样一个个凸起来）。

```cpp
int my_variable_name;   // 合乎惯例（下划线 / snake_case）
int my_function_name(); // 合乎惯例（下划线 / snake_case）

int myVariableName;     // 合乎惯例（驼峰 / camelCase）
int myFunctionName();   // 合乎惯例（驼峰 / camelCase）

int my variable name;   // 非法（不允许出现空白字符）
int my function name(); // 非法（不允许出现空白字符）

int MyVariableName;     // 不合惯例（应以小写字母开头）
int MyFunctionName();   // 不合惯例（应以小写字母开头）
```

在本教程里，我们通常使用**驼峰式**，因为它读起来更清晰（在代码密集的地方，下划线容易被看成空格）。但两种风格都很常见——C++ 标准库对变量和函数都使用**下划线风格**。你也常会看到**混用**：变量用下划线，函数用驼峰。

如果你是在别人的代码里工作，一般来说，**匹配你所在项目的风格**比死板地遵循上面这些命名建议更合适。

> **最佳实践**
>
> **在已有项目里，跟随该项目的命名约定**（即便它不符合现代最佳实践）。**写新程序时，使用现代最佳实践。**

### 3）不要以下划线开头命名

**避免用下划线作为标识符的开头**。虽然这在语法上是合法的，但这类名字通常**保留给操作系统、库、和编译器使用**。

### 4）名字要能清楚表达含义

标识符的名字应当能**清楚地说明它所保存的值意味着什么**（尤其在单位不明显的时候）。命名时应当让**一个完全不了解你代码的人**也能尽快看懂它。三个月后你再回头看自己的程序时，你也早就忘了它是怎么工作的——那时候你会感谢过去的自己起了有意义的变量名。

不过，**给一个平凡的标识符起一个过于复杂的名字**，对代码可读性的伤害，几乎和**给一个重要的标识符起一个含糊的名字**一样大。一个不错的经验法则是：**让标识符名字的长度与它的“具体性”和“可见范围”成正比**。也就是说：

- **只在几条语句内存在**的标识符（例如短函数体里的局部变量）可以用**较短**的名字；
- **在任何地方都能访问**的标识符适合用**较长**的名字；
- **表示不特定含义的数值**（例如用户随便输入的一个数）可以用**较短**的名字；
- **表示某个具体含义的值**（例如“裤腿内侧长度，单位毫米”）应当用**较长**的名字。

| 名字                         | 好 / 差 | 说明                                   |
| :--------------------------- | :------ | :------------------------------------- |
| `int ccount`                 | ❌ 差    | `count` 前面这个 `c` 是啥意思？        |
| `int customerCount`          | ✅ 好    | 一目了然：我们在数什么                 |
| `int i`                      | 视情况  | 用途简单时可以，否则不好               |
| `int index`                  | 视情况  | 如果“索引什么”很明显就可以             |
| `int totalScore`             | 视情况  | 如果只有一件事在打分就可以，否则太含糊 |
| `int _count`                 | ❌ 差    | 不要以下划线开头                       |
| `int count`                  | 视情况  | 如果“数什么”一目了然就可以             |
| `int data`                   | ❌ 差    | 什么样的数据？                         |
| `int time`                   | ❌ 差    | 秒、分、还是小时？                     |
| `int minutesElapsed`         | 视情况  | 如果“从何时算起”很明显就可以           |
| `int x1, x2`                 | 视情况  | 用途简单时可以，否则不好               |
| `int userinput1, userinput2` | ❌ 差    | 名字长又相似，很难区分                 |
| `int numApples`              | ✅ 好    | 描述性强                               |
| `int monstersKilled`         | ✅ 好    | 描述性强                               |

### 5）避免使用缩写

**避免使用缩写**，除非它是**常见且无歧义**的（例如 `num`、`cm`、`idx`）。

> **关键洞察**
>
> **代码被阅读的次数远多于被书写的次数**。所以你在写代码时省下来的每一点时间，都会变成**未来你以及每一位读者**在阅读时浪费掉的时间。**如果你想更快地写代码，请使用编辑器的自动补全功能**，而不是靠缩写省几个字符。

### 6）必要时用注释补充说明

对于**变量声明**，用注释说明变量的用途或其他不明显的信息通常很有帮助。举例来说，假设我们声明了一个变量，用来保存一段文本中的字符数。那么 `"Hello World!"` 这段文本到底是 10、11 还是 12 个字符呢？这取决于我们是否把空格和标点也算进去。与其把变量名起成又臭又长的 `numCharsIncludingWhitespaceAndPunctuation`，不如**在声明行上方或旁边加一条注释**，帮助读者理清含义：

```cpp
// 一段文本中包含的字符数量，含空白与标点
int numChars {};
```
