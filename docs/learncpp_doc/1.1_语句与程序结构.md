# 1.1 — 语句与程序结构

---

## 语句（Statements）

计算机程序是一系列告诉计算机该做什么的**指令**。而**语句（statement）**是一种能让程序执行某种操作的指令。

语句是 C++ 程序中最常见的指令类型，因为它是 C++ 中**最小的独立计算单元**。从这个角度来说，它就像自然语言中的**句子**。当我们想向别人表达一个想法时，我们通常用完整的句子（而不是零散的词或音节）。在 C++ 中，当我们想让程序做某件事时，我们通常也是通过写语句来实现的。

C++ 中的大多数（但不是全部）语句以**分号**结尾。如果你看到某一行以分号结尾，那它很可能就是一条语句。

在 C++ 这样的高级语言中，**一条语句可能会被编译成很多条机器语言指令**。

> C++ 中有很多种语句：
>
> - 声明语句（Declaration statements）
> - 跳转语句（Jump statements）
> - 表达式语句（Expression statements）
> - 复合语句（Compound statements）
> - 选择语句（Selection statements，即条件语句）
> - 迭代语句（Iteration statements，即循环）
> - try 块（Try blocks）

## 函数与 main 函数

在 C++ 中，语句通常被组织成一个个单元，称为**函数（function）**。一个函数是一组会**按顺序**（从上到下）依次执行的语句。等你学会写自己的程序后，你就能创建自己的函数，并按自己的方式随意组合语句（后续课程会讲到）。

> **规则**
>
> 每一个 C++ 程序都必须有一个特殊的函数，名字必须是 `main`（**全部小写**）。

程序运行时，会**按顺序**执行 `main` 函数中的语句。程序通常在 `main` 函数中最后一条语句执行完后终止（尽管在某些情况下程序可能提前中止，或在结束后做一些清理工作）。

函数一般是被写来完成某个特定任务的。例如：

- 一个叫 `max` 的函数，可能包含用来判断两个数中哪个更大的语句；
- 一个叫 `calculateGrade` 的函数，可能用来根据学生的考试分数计算成绩；
- 一个叫 `printEmployee` 的函数，可能用于把员工信息打印到控制台。

我们很快就会更多地讨论函数，因为它们是程序中最常用的组织工具。

> **命名约定**
>
> 讨论函数时，通常会在函数名后面加上一对括号作为简写。例如，写作 `main()` 或 `doSomething()`，就是指名为 `main` 或 `doSomething` 的函数。这样可以把函数与其它有名字的东西（例如变量）区分开来，而不用每次都写“函数”这两个字。
>
> 在编程中，一个函数（或对象、类型、模板等）的名字被称为**标识符（identifier）**。

## 字符与文本

最早的计算机主要是为数学计算和数据处理设计的。随着硬件的发展、网络的普及以及消费级软件的演进，计算机也逐渐成为**书面交流**的重要工具。

在书面语言中，最基本的交流单位是**字符（character）**。简单说，字符就是一个书面符号或标记，比如字母、数字、标点符号或数学符号。当我们在键盘上按下一个字母或数字键时，就会产生一个字符，并可以显示在屏幕上。下面这些都是字符：`a`、`2`、`$` 和 `=`。

很多情况下，例如写词语或句子时，我们会用到不止一个字符。**由多个字符组成的序列称为文本（text）**，在编程中也叫**字符串（string）**。

> **命名约定**
>
> 按照惯例，“文本”一词也常被用来指**纯文本（plain text）**，即只包含标准键盘上出现的字符、没有特殊格式或样式的文本。例如，纯文本不能表示加粗的单词，因为加粗需要样式。
>
> 我们的 C++ 程序就是以纯文本形式编写的。

> **进阶阅读**
>
> 计算机中还有一种叫做**控制字符（control character）**的字符。它们对计算机系统有特殊含义，但要么不用于显示，要么显示为非单个可见符号的形式。常见的控制字符例如：“Escape”（不会显示任何东西）、“Tab”（显示为若干空格）、“Backspace”（删除前一个字符）。

> **相关内容**
>
> - 关于字符与文本的输出，参见 1.5 —— iostream 简介：`cout`、`cin` 与 `endl`。
> - 关于字符（包括控制字符）的更详细介绍，参见 4.11 —— Chars。

## 拆解 “Hello world!” 程序

现在你已经初步了解了什么是语句、什么是函数，让我们回到“Hello world”程序，从更高的层面看看每一行做了什么。

```cpp
#include <iostream>
using namespace std;

int main()
{
   cout << "Hello world!";
   return 0;
}
```

- **第 1 行**是一种特殊的行，叫做**预处理指令（preprocessor directive）**。`#include` 预处理指令表示我们希望使用 `iostream` 库的内容，这个库是 C++ 标准库中用于**从控制台读写文本**的部分。我们需要这一行，才能在第 5 行使用 `cout`。如果没有这一行，第 5 行会报编译错误，因为编译器不知道 `cout` 是什么。

- **第 2 行**是空行，会被编译器忽略。这一行只是为了让程序对人类来说**更易读**（把预处理指令和后面的部分分开）。

- **第 3 行**告诉编译器：我们要定义一个函数，它的名字（标识符）是 `main`。前面提到过，每个 C++ 程序必须有一个 `main` 函数，否则将无法链接成功。这个函数返回值的类型是 `int`（整数）。

- **第 4 行和第 7 行**告诉编译器哪些行属于 `main` 函数。第 4 行的左大括号 `{` 与第 7 行的右大括号 `}` 之间的所有内容都属于 `main` 函数，这被称为**函数体（function body）**。

- **第 5 行**是 `main` 函数中的第一条语句，也是程序运行时第一条会被执行的语句。`std::cout`（意思是 “character output”，即字符输出）和 `<<` 运算符可以让我们向控制台显示信息。这里，我们显示了文本 `Hello world!`。这条语句产生了程序的可见输出。

- **第 6 行**是一条**返回语句（return statement）**。当一个可执行程序运行结束时，会向操作系统返回一个值，用来表示它是否成功运行。这里 `return 0;` 表示向操作系统返回整数 `0`，意思是“一切正常！”。这也是程序中最后一条执行的语句。

我们后面写的所有程序，都会遵循这个大致模板或它的某种变体。

你可以自己编译并运行这个程序，你会看到控制台输出：

```
Hello world!
```

## 语法与语法错误

在英语中，句子是按照特定的**语法规则**构造的，这些规则你在学校的英语课上应该学过。例如，普通句子以句号结尾。**描述哪些词（和标点）能以怎样的方式组合成合法句子的规则**称为**语法（syntax）**。例如，“My house painted is blue” 是一个语法错误，因为词语顺序不合规。另一个著名的例子是 “All your base are belong to us!”。

C++ 编程语言也有自己的语法，规定了程序中的各种元素必须**如何书写和排列**才算合法。当你编译程序时，编译器负责检查你的程序是否遵循这些语法规则。如果你的程序偏离了 C++ 的语法，编译器会中止编译并报告一个 **语法错误（syntax error）**。

和允许大量歧义的英语不同，C++ 的语法规则被**严格定义、严格执行**。语法错误非常常见，好消息是这类错误通常很容易发现和修复，因为编译器一般会直接指出出错位置。只有当所有语法错误都被解决后，程序的编译才会成功完成。

由于大多数语句的语法都要求以分号结尾，我们来看看如果去掉 “Hello world” 程序第 5 行的分号会发生什么：

```cpp
#include <iostream>
using namespace std;

int main()
{
   cout << "Hello world!"
   return 0;
}
```

你可以尝试编译这段有问题的程序。

用 `vscode` 编译时，会得到如下错误信息：

```
6:26: error: expected ';' before 'return'
```

`vscode` 告诉我们：在第 6 行第 26 个字符处，语法规则要求有一个分号，但我们并没有写。因此编译因错误而中止
