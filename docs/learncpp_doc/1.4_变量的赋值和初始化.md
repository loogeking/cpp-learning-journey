# 1.4 — 变量的赋值与初始化

---

在上一节 [1.3 — 对象与变量简介](1.3_对象和变量简介.md) 中，我们讲解了如何**定义**一个可以用来存储值的变量。本节中，我们将学习**如何真正把值放进变量里**。

先复习一下——下面这段短程序先定义了一个名为 `x` 的整型变量，然后又定义了两个整型变量 `y` 和 `z`：

```cpp
int main()
{
    int x;    // 定义一个整型变量 x（推荐写法）
    int y, z; // 定义两个整型变量 y 和 z

    return 0;
}
```

再提醒一次：**推荐每行只定义一个变量**。稍后我们会回过头再讨论一次定义多个变量的情况。

---

## 变量赋值（Variable assignment）

在变量被定义之后，你可以（用另一条语句）通过 `=` 操作符给它一个值。这个过程被称为**赋值（assignment）**，而 `=` 被称为**赋值操作符（assignment operator）**。

```cpp
int width; // 定义一个整型变量 width
width = 5; // 把值 5 赋给变量 width

// 现在 width 的值就是 5
```

默认情况下，赋值会**把 `=` 右边的值拷贝到左边的变量中**，这被称为**拷贝赋值（copy-assignment）**。

变量一旦有了值，就可以通过 `std::cout` 和 `<<` 操作符把它打印出来。

只要我们想改变某个变量当前所保存的值，就可以使用赋值。下面这个例子中我们赋值了两次：

```cpp
#include <iostream>

int main()
{
    int width; // 定义一个变量 width
    width = 5; // 把值 5 拷贝赋值给 width

    std::cout << width; // 输出 5

    width = 7; // 把 width 中保存的值改为 7

    std::cout << width; // 输出 7

    return 0;
}
```

这个程序会输出：

```
57
```

程序运行时，会从 `main` 函数的顶部开始按顺序执行。首先为变量 `width` 分配内存；接着把 5 赋给 `width`；此时输出 `width` 会打印 `5`；然后把 7 赋给 `width`，之前的值（5）就被覆盖掉了；再次输出 `width`，就打印 `7`。

**普通变量一次只能保存一个值。**

> **警告**
>
> 初学者最常犯的错误之一，就是把**赋值操作符 `=`** 和**相等操作符 `==`** 搞混。
>
> - `=` 用于把一个值赋给变量；
> - `==` 用于判断两个操作数的值是否相等。

---

## 变量的初始化（Variable initialization）

赋值有一个缺点：给一个**刚定义好**的对象赋值需要**两条语句**——一条定义变量，一条给它赋值。

这两步是可以合并的。**当我们定义一个对象时，可以顺便给它一个初始值。**给对象指定初始值的过程叫做**初始化（initialization）**，用来初始化对象的语法叫做**初始化器（initializer）**。口语中，这个初始值本身也常被称为“初始化器”。

例如，下面这条语句**既定义了一个 `int` 类型的变量 `width`，又用值 `5` 对其进行了初始化**：

```cpp
#include <iostream>

int main()
{
    int width { 5 };    // 定义变量 width，并用初始值 5 进行初始化
    std::cout << width; // 输出 5

    return 0;
}
```

在上面对 `width` 的初始化中，`{ 5 }` 是初始化器，`5` 是初始值。

> **关键洞察**
>
> **初始化（initialization）就是给变量提供一个初始值。** 记忆方法：**initial-ization**（初始 + 化）。

---

## 初始化的几种形式

与赋值（一般来说很直白）不同，C++ 中的**初始化其实相当复杂**。所以我们先给出一个**简化版本**帮你入门。

C++ 中常见的初始化形式有 **5** 种：

```cpp
int a;         // 默认初始化（无初始化器）

// 传统初始化形式：
int b = 5;     // 拷贝初始化（等号后面写初始值）
int c ( 6 );   // 直接初始化（初始值写在圆括号里）

// 现代初始化形式（推荐）：
int d { 7 };   // 直接列表初始化（初始值写在花括号里）
int e {};      // 值初始化（空花括号）
```

你可能会在别处见到这几种写法的不同空格风格（例如 `int b=5;`、`int c(6);`、`int d{7};`、`int e{};`）。是否为了可读性多加空格，纯属个人偏好。

从 C++17 起，**拷贝初始化、直接初始化、直接列表初始化在大多数情况下表现是一致的**。它们之间**最重要的不同点**我们会在下面单独说明。

> **相关内容**
>
> 关于拷贝初始化、直接初始化、列表初始化之间的其它差异，我们会在 14.15 — 类的初始化与拷贝省略 中讲到。

> **进阶阅读**
>
> 其他形式的初始化还包括：
>
> - 聚合初始化（见 13.8 — 结构体聚合初始化）
> - 拷贝列表初始化（下文讨论）
> - 引用初始化（见 12.3 — 左值引用）
> - 静态初始化、常量初始化和动态初始化（见 7.8 — 为什么（非常量）全局变量是万恶之源）
> - 零初始化（下文讨论）

## 默认初始化（Default-initialization）

如果**没有提供任何初始化器**（比如上面的变量 `a`），这种情况叫做**默认初始化**。在很多情况下，默认初始化**根本什么都不做**，会让变量保有一个**不可预测的值（indeterminate value，有时也叫“垃圾值 / garbage value”）**。

关于这种情况带来的问题，我们会在 1.6 — 未初始化变量与未定义行为 中进一步讨论。

---

## 拷贝初始化（Copy-initialization）

当我们在**等号**后面提供初始值时，这被称为**拷贝初始化**。这种形式是从 C 语言继承来的：

```cpp
int width = 5; // 用值 5 拷贝初始化变量 width
```

和拷贝赋值类似，它会把等号右边的值拷贝到左边正在被创建的变量中。上面这段代码里，`width` 会被初始化为 5。

拷贝初始化在现代 C++ 中一度不太受欢迎，因为对某些复杂类型来说，它比其它初始化形式**效率更低**。不过 C++17 修正了大部分这类问题，拷贝初始化因此又有了新的支持者。你也会在旧代码里看到它（尤其是从 C 移植过来的代码），有些开发者也只是觉得它看起来更自然、更易读。

---

## 直接初始化（Direct-initialization）

当我们在**圆括号**里提供初始值时，这被称为**直接初始化**：

```cpp
int width ( 5 ); // 用值 5 直接初始化变量 width
```

直接初始化最初是为了让**复杂对象**（类类型对象，后面章节会讲）能被更高效地初始化。和拷贝初始化一样，直接初始化在现代 C++ 中一度不受欢迎，主要因为被**直接列表初始化**取代了。不过，直接列表初始化本身也有一些坑，所以在某些场景中直接初始化又重新被使用起来。

---

## 列表初始化（List-initialization）

**在现代 C++ 中，初始化对象的推荐方式是使用花括号形式的初始化**，这被称为**列表初始化（list-initialization）**，也叫**统一初始化（uniform initialization）** 或**花括号初始化（brace initialization）**。

列表初始化有两种形式：

```cpp
int width { 5 };    // 直接列表初始化：把 5 初始化给 width（推荐）
int height = { 6 }; // 拷贝列表初始化：把 6 初始化给 height（很少使用）
```

在 C++11 之前，某些类型必须使用拷贝初始化，另一些类型又必须使用直接初始化。拷贝初始化容易和拷贝赋值混淆（因为都用 `=`），直接初始化又容易和函数调用之类的操作混淆（因为都用圆括号）。

列表初始化的引入，就是为了提供一种**几乎在所有情况下都能用、行为一致、语法上没有歧义**的初始化方式，让人一眼就能看出这是在初始化对象。

> **关键洞察**
>
> **看到花括号，就知道正在进行的是列表初始化。**

此外，列表初始化还提供了**用一组值来初始化对象**（而不只是一个值）的方式，这也是它被称为“列表”初始化的原因。我们会在 16.2 — std::vector 简介与列表构造函数 中看到相关例子。

---

## 列表初始化禁止“窄化转换（narrowing conversions）”

对 C++ 新手来说，列表初始化的一个主要好处是：**它禁止“窄化转换”**。也就是说，如果你试图用一个**变量装不下的值**去列表初始化它，**编译器必须给出诊断信息**（编译错误或警告）。例如：

```cpp
int main()
{
    // 整数只能保存不含小数部分的值。
    // 用带小数的 4.5 初始化 int，需要编译器把 4.5 转成 int 能表示的值。
    // 这种转换是“窄化转换”，因为小数部分会丢失。

    int w1 { 4.5 }; // 编译错误：列表初始化不允许窄化转换

    int w2 = 4.5;   // 可以编译：w2 被拷贝初始化为 4
    int w3 (4.5);   // 可以编译：w3 被直接初始化为 4

    return 0;
}
```

上面第 7 行，我们用含小数（`.5`）的值去列表初始化一个整型变量，这是窄化转换，所以**编译器必须报错**。

而拷贝初始化（第 9 行）和直接初始化（第 10 行）会**默默地丢掉小数部分**，把变量初始化为 `4`（这很可能不是我们想要的结果）。编译器**可能**会为此发出警告（因为丢数据通常不是好事），但也可能不会警告。

需要注意的是，**这个“禁止窄化转换”的限制只作用于列表初始化本身**，对之后的赋值不生效：

```cpp
int main()
{
    int w1 { 4.5 }; // 编译错误：列表初始化不允许把 4.5 窄化为 4

    w1 = 4.5;       // 没问题：拷贝赋值允许把 4.5 窄化为 4

    return 0;
}
```

## 值初始化（Value-initialization）与零初始化（Zero-initialization）

当我们用**一对空花括号**来初始化变量时，会发生一种特殊的列表初始化，叫做**值初始化（value-initialization）**。大多数情况下，值初始化会**隐式地把变量初始化为 0**（或者对某个类型来说“最接近 0 的值”）。当这种“清零”发生时，也叫**零初始化（zero-initialization）**。

```cpp
int width {}; // 值初始化 / 零初始化为 0
```

> **进阶阅读**
>
> 对于**类类型**来说，值初始化（以及默认初始化）可能会用类**预先定义好的默认值**来初始化对象，而这个默认值**不一定是 0**。

---

## 在现代 C++ 中，列表初始化是首选形式

列表初始化（包括值初始化）通常优于其它初始化形式，原因是：

- 它在大多数情况下都能用（因此更一致）；
- 它禁止窄化转换（通常我们并不希望发生窄化）；
- 它支持使用一组值来初始化对象（后面课程会讲）。

> **最佳实践**
>
> **优先使用直接列表初始化或值初始化来初始化变量。**

**Q: 什么时候用 `{ 0 }`，什么时候用 `{}`？**

- 当你**要用**这个初始值的时候，用**直接列表初始化**：

  ```cpp
  int x { 0 };    // 直接列表初始化为 0
  std::cout << x; // 我们确实在使用这个 0
  ```

- 当这个初始值只是**临时的、马上会被替换**时，用**值初始化**：

  ```cpp
  int x {};      // 值初始化
  std::cin >> x; // 马上就要覆盖掉，写一个显式的 0 意义不大
  ```

---

## 实例化（Instantiation）

**实例化（instantiation）** 是一个听起来很花哨的词，意思其实是：**一个变量被创建（分配了内存）并被初始化了**（包括默认初始化）。被实例化的对象有时也叫**实例（instance）**。这个术语最常用在类类型对象上，但偶尔也用来描述其它类型的对象。

---

## 一次初始化多个变量

在上一节我们提到过，可以在同一条语句中定义多个类型相同的变量，各变量名用逗号分隔：

```cpp
int a, b; // 创建变量 a 和 b，但都不初始化
```

我们也提到**最佳做法是完全不要这样写**。不过既然你可能在别处见到这种代码，还是有必要多说几句——哪怕只是加深你**为什么应该避免**这样写的印象。

你可以同时初始化定义在同一行的多个变量：

```cpp
int a = 5, b = 6;          // 拷贝初始化
int c ( 7 ), d ( 8 );      // 直接初始化
int e { 9 }, f { 10 };     // 直接列表初始化
int i {}, j {};            // 值初始化
```

不幸的是，这里有一个常见坑：程序员**错以为**一条初始化语句可以同时初始化两个变量：

```cpp
int a, b = 5;     // 错的：a 没有被初始化为 5！
int a = 5, b = 5; // 对的：a 和 b 都被初始化为 5
```

在上面第一条语句中，变量 `a` **没有被初始化**，编译器**可能**会警告，也可能不会。如果没有警告，那么这就是一种让程序**偶尔崩溃或结果时对时错**的绝佳来源。关于使用未初始化变量会发生什么，我们很快就会讲到。

要记住这条写法是错的，只需记住一点：**每个变量只能被它自己的初始化器初始化**：

```cpp
int a = 4, b = 5; // 对的：a 和 b 都有各自的初始化器
int a, b = 5;     // 错的：a 没有自己的初始化器
```
