# 2.4 — 函数形参与实参简介

---

在上一节中，我们学到函数可以**把值返回给调用方**。我们据此写出了一个模块化的 `getValueFromUser` 函数，用在了这个程序里：

```cpp
#include <iostream>

int getValueFromUser()
{
    std::cout << "Enter an integer: ";
    int input{};
    std::cin >> input;

    return input;
}

int main()
{
    int num { getValueFromUser() };

    std::cout << num << " doubled is: " << num * 2 << '\n';

    return 0;
}
```

那如果我们**也想把“输出那一行”单独封装成一个函数**呢？你可能会写成这样：

```cpp
#include <iostream>

int getValueFromUser()
{
    std::cout << "Enter an integer: ";
    int input{};
    std::cin >> input;

    return input;
}

// 这个函数编译不过
void printDouble()
{
    std::cout << num << " doubled is: " << num * 2 << '\n';
}

int main()
{
    int num { getValueFromUser() };

    printDouble();

    return 0;
}
```

它编译不过，因为 `printDouble` **根本不知道 `num` 是谁**。你可能又想在 `printDouble` 里面自己定义一个 `num`：

```cpp
void printDouble()
{
    int num{}; // 我们新加的一行
    std::cout << num << " doubled is: " << num * 2 << '\n';
}
```

这样虽然能编译过，但**程序仍然是错的**（它会永远输出 `0 doubled is: 0`）。**问题的根源在于**：`printDouble` **无法获取用户输入的那个值**。

我们需要一种方式，**把 `num` 这个值“传递”给 `printDouble`**，让它在函数体里用得上这个值。

---

## 函数的形参（parameter）与实参（argument）

很多时候，我们需要**把信息传给被调用的函数**，让它有数据可以处理。比如你想写一个“把两个数相加”的函数，就得**告诉它要加哪两个数**——否则它怎么知道要加什么？这正是通过**形参**和**实参**来实现的。

**形参（function parameter）** 是**写在函数头里的一种特殊变量**。它几乎和函数内部定义的变量一样，但有一个关键区别：**形参的初始值是由函数的调用方提供的**。

形参写在**函数名后面的一对圆括号里**，多个形参之间用**逗号**分隔。

看看几个不同形参数量的函数：

```cpp
// 这个函数没有形参
// 它不依赖调用方提供任何数据
void doPrint()
{
    std::cout << "In doPrint()\n";
}

// 这个函数有 1 个 int 形参 x
// x 的值由调用方提供
void printValue(int x)
{
    std::cout << x << '\n';
}

// 这个函数有 2 个 int 形参：x 和 y
// x 和 y 的值都由调用方提供
int add(int x, int y)
{
    return x + y;
}
```

**实参（argument）** 就是**调用函数时传给它的具体值**：

```cpp
doPrint();      // 这次调用没有实参
printValue(6);  // 6 是传给 printValue() 的实参
add(2, 3);      // 2 和 3 是传给 add() 的实参
```

多个实参之间同样用**逗号**分隔。

---

## 形参与实参是如何配合的？

**函数被调用时**，函数的所有形参会以变量的形式被**创建**，然后每个实参的值会通过**拷贝初始化**赋给对应的形参。这个过程叫做 **按值传递（pass by value）**。**采用按值传递方式的形参称为 “值形参（value parameter）”。**

例如：

```cpp
#include <iostream>

// 这个函数有 2 个 int 形参：x 和 y
// x 和 y 的值由调用方传入
void printValues(int x, int y)
{
    std::cout << x << '\n';
    std::cout << y << '\n';
}

int main()
{
    printValues(6, 7); // 传了 2 个实参：6 和 7

    return 0;
}
```

当 `printValues` 被以 `6` 和 `7` 为实参调用时：形参 `x` 被创建、并用值 `6` 初始化；形参 `y` 被创建、并用值 `7` 初始化。

输出：

```
6
7
```

要注意：**实参的个数一般必须与形参的个数一致**，否则编译器会报错。传给函数的实参可以是任何合法的**表达式**（因为实参本质上就是形参的初始化器，而初始化器可以是任何合法表达式）。

## 修好我们那个有问题的程序

现在我们有工具修好开头那段程序了：

```cpp
#include <iostream>

int getValueFromUser()
{
    std::cout << "Enter an integer: ";
    int input{};
    std::cin >> input;

    return input;
}

void printDouble(int value) // 这个函数现在多了一个 int 形参
{
    std::cout << value << " doubled is: " << value * 2 << '\n';
}

int main()
{
    int num { getValueFromUser() };

    printDouble(num);

    return 0;
}
```

在这段程序里：`num` 首先被用户输入的值初始化；接着 `printDouble` 被调用，实参 `num` 的值被拷贝进 `printDouble` 的形参 `value`；然后 `printDouble` 就使用它自己那份 `value` 完成输出。

---

## 用返回值直接作为实参

在上面的程序里，`num` 其实**只被用过一次**：把 `getValueFromUser()` 的返回值，转手交给 `printDouble()` 的实参。

我们可以稍微简化一下：

```cpp
#include <iostream>

int getValueFromUser()
{
    std::cout << "Enter an integer: ";
    int input{};
    std::cin >> input;

    return input;
}

void printDouble(int value)
{
    std::cout << value << " doubled is: " << value * 2 << '\n';
}

int main()
{
    printDouble(getValueFromUser());

    return 0;
}
```

这样，我们**直接把 `getValueFromUser` 的返回值作为 `printDouble` 的实参**！

这种写法更简洁，同时也**清楚地表明**：用户输入的这个值“除了拿去 double 一下、别的都不做”。但如果你觉得这种“紧凑写法”读起来有点吃力，**保留原来带 `num` 的版本也完全 OK**。

---

## 形参 + 返回值：函数最有用的组合

结合**形参**和**返回值**，我们可以写出这样的函数：**接收输入 → 做点计算 → 把结果返回给调用方**。

一个很简单的例子——把两个数相加并返回结果：

```cpp
#include <iostream>

// add() 接收两个整数作为形参，返回它们的和
// x 和 y 的值由调用它的地方决定
int add(int x, int y)
{
    return x + y;
}

// main 没有形参
int main()
{
    std::cout << add(4, 5) << '\n'; // 把实参 4 和 5 传给 add()
    return 0;
}
```

执行从 `main` 顶部开始。求值 `add(4, 5)` 时，`add` 被调用：形参 `x` 用 `4` 初始化，形参 `y` 用 `5` 初始化。`add` 中的 `return x + y;` 把 `x + y` 求值为 `9`，然后返回给 `main`，接着 `std::cout` 把 `9` 打印到控制台。

输出：

```
9
```

## 更多例子

再看几个函数调用的例子：

```cpp
#include <iostream>

int add(int x, int y)
{
    return x + y;
}

int multiply(int z, int w)
{
    return z * w;
}

int main()
{
    std::cout << add(4, 5) << '\n';           // add() 里 x=4, y=5, 结果为 9
    std::cout << add(1 + 2, 3 * 4) << '\n';   // add() 里 x=3, y=12, 结果为 15

    int a{ 5 };
    std::cout << add(a, a) << '\n';           // 相当于 add(5, 5) = 10

    std::cout << add(1, multiply(2, 3)) << '\n'; // 相当于 1 + (2 * 3) = 7
    std::cout << add(1, add(2, 3)) << '\n';      // 相当于 1 + (2 + 3) = 6

    return 0;
}
```

程序输出：

```
9
15
10
7
6
```

第一条语句很直白。

第二条语句中，实参本身是**表达式**——它们会**先求值再传入**。`1 + 2` 求值为 `3`，拷给形参 `x`；`3 * 4` 求值为 `12`，拷给形参 `y`；`add(3, 12)` 得到 `15`。

接下来这一对也不难：

```cpp
int a{ 5 };
std::cout << add(a, a) << '\n'; // 相当于 add(5, 5)
```

`a` 的值 `5` 被拷贝到形参 `x` 和 `y`，`add(5, 5)` 返回 `10`。

看看这段稍微“嵌套”的：

```cpp
std::cout << add(1, multiply(2, 3)) << '\n'; // 相当于 1 + (2 * 3)
```

要执行 `add`，程序需要知道形参 `x` 和 `y` 是多少。`x` 直接是字面量 `1`；`y` 则需要先求 `multiply(2, 3)`。`multiply` 里 `z = 2`、`w = 3`，返回 `6`。这个 `6` 就成了 `add` 的形参 `y`。`add(1, 6)` 返回 `7`，被送去 `std::cout` 打印。

一句话概括：`add(1, multiply(2, 3))` → `add(1, 6)` → `7`。

下一个稍微“套娃”一点，`add` 的其中一个实参又是一次 `add` 调用：

```cpp
std::cout << add(1, add(2, 3)) << '\n'; // 相当于 1 + (2 + 3)
```

处理方式是一样的：先求 `add(2, 3)` 得到 `5`，然后求 `add(1, 5)` 得到 `6`，最后 `std::cout` 打印。

一句话概括：`add(1, add(2, 3))` → `add(1, 5)` → `6`。

## 未使用的形参 与 未命名的形参

有时你会遇到**形参在函数体里根本没被用到**的情况，这类形参叫做 **未使用形参（unreferenced parameter）**。

一个简单例子：

```cpp
void doSomething(int count) // 警告：形参 count 未被使用
{
    // 以前会用到 count，现在不用了
}

int main()
{
    doSomething(4);

    return 0;
}
```

和未使用的局部变量一样，编译器很可能会**警告** `count` 定义了却没使用。

**在函数定义中，形参的名字其实是可选的。** 因此，当你**需要保留形参，但函数体里用不到它**时，可以**直接省略形参名**。**没有名字的形参就叫“未命名形参（unnamed parameter）”**：

```cpp
void doSomething(int) // OK：未命名形参不会触发警告
{
}
```

**Google C++ 风格指南**建议用**注释**说明这个未命名形参原本代表什么：

```cpp
void doSomething(int /*count*/)
{
}
```

> **作者笔记**
>
> 你可能会想：为什么要写一个“形参的值根本不用”的函数？最常见的情况类似这样：
>
> 假设你原本有一个带一个形参的函数，后来函数被更新过，这个形参的值不再被使用。如果你**直接把这个不再需要的形参删掉**，那所有已经存在的调用都会**编译不过**（因为调用点传的实参数量比函数能接受的多了）。要修好，就得**找到所有调用点并删掉多余的实参**——工作量可能很大（还得回归测试一大堆东西），甚至有些调用你根本控制不了（比如别人代码里也在调用你的函数）。所以更稳妥的做法就是：**保留这个形参，但让它什么也不做**。

> **进阶阅读**
>
> 还有一些其它场景会用到未命名形参，例如：
>
> - `++` 和 `--` 有前置和后置两种形式（`++foo` vs `foo++`）。**未命名形参**被用来**区分某个运算符重载的是前置版本还是后置版本**。这一点我们会在 21.8 — 重载自增自减运算符 中讲到。
> - 当我们需要**从类型模板参数的类型（而不是它的值）**中判断某些信息时。

> **作者笔记**
>
> 如果“未命名形参”现在对你还有点抽象，别担心。以后遇到具体场景时，我们会再解释它们的用处。

> **最佳实践**
>
> 如果一个形参在函数体里没有被使用，**就不要给它起名字**。你可以选择用注释注明它原本代表什么。

---

## 小结

**形参**和**返回值**是**让函数可复用**的核心机制：它们让我们能写出这样的函数——**接收输入数据 → 处理 → 把结果返回给调用方**，而**无需在写函数时就知道具体的输入是什么、输出是什么**。
