# 1.5 — iostream 简介：cout、cin 与 endl

---

在这节课里，我们会更深入地讲解 `std::cout`——我们在 “Hello world!” 程序里用它把文字输出到控制台。同时，我们还会学习如何**从用户那里获取输入**，让程序更具交互性。

---

## 输入 / 输出库（iostream 库）

**输入 / 输出库（I/O library）** 是 C++ 标准库的一部分，专门处理基本的输入和输出。我们会使用其中的功能来**从键盘读取输入**、**向控制台输出数据**。`iostream` 中的 `io` 就是 “input/output（输入/输出）” 的意思。

要使用 `iostream` 中定义的功能，就需要在使用它的任何代码文件顶部**包含（include）** `iostream` 头文件：

```cpp
#include <iostream>

// 后面的代码可以使用 iostream 中的功能
```

---

## std::cout

`iostream` 库提供了一些预定义变量供我们使用。其中最有用的之一就是 **`std::cout`**，它可以把数据发送到控制台，以文本形式打印出来。`cout` 是 **“character output（字符输出）”** 的缩写。

再回顾一下 Hello world 程序：

```cpp
#include <iostream> // 为了使用 std::cout

int main()
{
    std::cout << "Hello world!"; // 把 Hello world! 打印到控制台

    return 0;
}
```

在这段程序里，我们包含了 `iostream` 从而能使用 `std::cout`。在 `main` 函数中，我们通过 `std::cout` 和**插入运算符 `<<`**，把文本 `Hello world!` 发送到控制台并打印出来。

`std::cout` 不仅可以打印文本，还可以打印数字：

```cpp
#include <iostream>

int main()
{
    std::cout << 4; // 把 4 打印到控制台

    return 0;
}
```

输出：

```
4
```

也可以用它来打印变量的值：

```cpp
#include <iostream>

int main()
{
    int x{ 5 }; // 定义整型变量 x，初始化为 5
    std::cout << x; // 把 x 的值（5）打印到控制台
    return 0;
}
```

输出：

```
5
```

**要在同一行输出多样东西**，可以在同一条语句中多次使用插入运算符 `<<`，把多段输出**连接（concatenate）**起来。例如：

```cpp
#include <iostream>

int main()
{
    std::cout << "Hello" << " world!";
    return 0;
}
```

这里我们用了两次 `<<`，先输出 `Hello`，再输出 ` world!`。程序输出：

```
Hello world!
```

> **小贴士**
>
> 可以把 `<<`（以及 `>>`）想象成一条**传送带**，把数据往箭头指示的方向运送。这里的数据被送到 `std::cout`，就会被输出到屏幕上。

再看一个例子，在同一条语句里同时打印文本和变量：

```cpp
#include <iostream>

int main()
{
    int x{ 5 };
    std::cout << "x is equal to: " << x;
    return 0;
}
```

输出：

```
x is equal to: 5
```

> **相关内容**
>
> 关于前缀 `std::` 的真正作用，我们会在 2.9 — 命名冲突与命名空间简介 中讨论。

## 使用 std::endl 输出一个换行

猜猜下面这段程序会输出什么？

```cpp
#include <iostream>

int main()
{
    std::cout << "Hi!";
    std::cout << "My name is Alex.";
    return 0;
}
```

结果可能出乎你的意料：

```
Hi!My name is Alex.
```

**分开的输出语句并不会在控制台产生分开的两行输出。**

要在控制台上换行输出，就需要**告诉控制台把光标移到下一行**，也就是输出一个**换行符（newline）**。换行符是**依赖操作系统**的一个字符（或字符序列），作用是把光标移到下一行的开头。

一种输出换行的方式是使用 `std::endl`（`endl` = “end line”，行结束）：

```cpp
#include <iostream>

int main()
{
    std::cout << "Hi!" << std::endl; // std::endl 让光标移到下一行
    std::cout << "My name is Alex." << std::endl;

    return 0;
}
```

输出：

```
Hi!
My name is Alex.
```

> **小贴士**
>
> 上面程序中，第二个 `std::endl` **严格来说不是必需的**，因为程序马上就结束了。但保留它有几个好处：
>
> 1. 表示这一行输出是一个 **“完整的表达”**，就像英文句子结尾的句号一样。
> 2. 把光标定位到下一行，将来添加输出（比如让程序再说一句 “bye!”）时，新内容会出现在你预期的位置，而不是紧接在上一行的末尾。
> 3. 在命令行运行可执行程序时，**有些操作系统不会在重新显示命令提示符前自动换行**。如果你的程序结束时光标不在新的一行，命令提示符可能会紧贴在你上一行输出后面出现。

> **最佳实践**
>
> **每当一行输出结束时，就输出一个换行。**

## std::cout 是带缓冲的（buffered）

想象一下你在游乐场排队坐过山车：乘客陆续到达并排队；一列过山车周期性地到站，让乘客上车（不超过它的载客上限）；等车装满或过了一段时间，车就带着这一批乘客出发；来不及上这一趟的乘客就等下一趟。

这个比喻和 C++ 中送往 `std::cout` 的输出通常是怎样处理的很像。程序里的语句请求把内容输出到控制台，但**这些输出通常并不会立即出现在控制台上**。相反，它们会**先“排队”**，被存进一块专门用来收集这些请求的内存区域——**缓冲区（buffer）**。缓冲区会**定期被“冲刷（flush）”**，也就是把里面积累的数据一次性送到它的目的地（这里就是控制台）。

> **作者笔记**
>
> 换个比喻，冲刷缓冲区有点像冲马桶：所有积累下来的“输出”一起被送去……它该去的地方。呃……

这也意味着：如果你的程序在缓冲区被 flush 之前就**崩溃、被中止、或被暂停**（例如调试时），那些还在缓冲区里的输出**将不会显示出来**。

> **关键洞察**
>
> 与“带缓冲输出”相对的是 **“无缓冲输出”**：每一次输出请求都会**直接送到输出设备**。

写入缓冲区通常很快，而把一批数据发送到输出设备则相对较慢。缓冲能**把多次输出请求合并成一批**，减少与输出设备打交道的次数，从而显著提升性能。

## std::endl 与 \n 的比较

使用 `std::endl` 往往效率不高，因为它实际上做了**两件事**：

1. 输出一个换行（把光标移到下一行）；
2. **flush（冲刷）缓冲区**（这一步是慢的）。

如果我们输出很多行文本、每行末尾都用 `std::endl`，就会**触发很多次 flush**，既慢又通常没有必要。

在向控制台输出文本时，我们通常**不需要自己显式地 flush 缓冲区**。C++ 的输出系统被设计成会**周期性地自动 flush**，让它自己 flush 更简单、也更高效。

要**只输出换行、但不 flush**，我们可以使用 `\n`（放在单引号或双引号中），编译器会把它解释成一个换行字符。`\n` 会把光标移到下一行，但**不会触发 flush**，所以通常性能更好。`\n` 也更短，还可以直接嵌到已有的双引号文本里。

下面这个例子演示了 `\n` 的几种用法：

```cpp
#include <iostream>

int main()
{
    int x{ 5 };
    std::cout << "x is equal to: " << x << '\n'; // 单独出现时用单引号（常规写法）
    std::cout << "Yep." << "\n";                 // 单独出现时也可以用双引号（不常规但可以）
    std::cout << "And that's all, folks!\n";     // 嵌在双引号文本里（常规写法）
    return 0;
}
```

输出：

```
x is equal to: 5
Yep.
And that's all, folks!
```

当 `\n` **不是嵌在**已有双引号文本中时（例如 `"hello\n"`），**惯例上写成单引号**：`'\n'`。

> **进阶阅读**
>
> 在 C++ 中，我们用**单引号**表示**单个字符**（如 `'a'`、`'$'`），用**双引号**表示**文本**（零个或多个字符）。
>
> 虽然 `'\n'` 在源代码中看起来是两个符号，但**编译器把它当作一个换行字符（LF，ASCII 值 10）** 来处理，因此惯例上写成单引号（除非它嵌在双引号文本里）。相关细节我们在 4.11 — Chars 中讨论。
>
> 当 `'\n'` 被输出时，负责输出的库会把这单个 LF 字符**翻译成当前操作系统对应的换行序列**。关于不同系统的换行约定，可以参考维基百科。

关于 `'\n'` 的更多细节，我们会在 4.11 — Chars 中详细讨论。

> **最佳实践**
>
> 在向控制台输出文本时，**优先使用 `\n` 而不是 `std::endl`**。

> **警告**
>
> `'\n'` 使用的是**反斜杠（\\）**（所有 C++ 特殊字符都是），**不是正斜杠**。
>
> 如果写成 `'/n'`，或在单引号里加入其它字符（如 `' \n'` 或 `'.\n'`），会导致**意想不到的行为**。例如 `std::cout << '/n';` 常常会打印出 `12142` 这样的东西，这很可能不是你想要的结果。

## std::cin

`std::cin` 是 `iostream` 库中另一个预定义变量。`std::cout` 用**插入运算符 `<<`**把数据输出到控制台；而 **`std::cin`（“character input”，字符输入）** 则用来**从键盘读取输入**。我们通常用**提取运算符 `>>`** 把输入数据存进一个变量（这样后面就可以使用它）。

```cpp
#include <iostream>

int main()
{
    std::cout << "Enter a number: "; // 提示用户输入一个数字

    int x{};       // 定义变量 x 用来保存用户输入（并值初始化）
    std::cin >> x; // 从键盘读入一个数字，存进变量 x

    std::cout << "You entered " << x << '\n';
    return 0;
}
```

自己动手编译并运行这个程序：第 5 行会打印 `Enter a number: `。执行到第 8 行时，程序会**等待你输入**。你输入一个数字并回车后，这个数字会被赋给变量 `x`；最后，第 10 行会输出 `You entered `，后面跟着你刚输入的数。

例如（输入 4）：

```
Enter a number: 4
You entered 4
```

这是从用户获取键盘输入的一种简单方式，后面很多例子里我们都会用到它。

> **小贴士**
>
> 输入一行数据时**不用**再自己输出 `'\n'`，因为用户必须按回车键提交输入，回车本身就已经把光标带到了下一行。

就像可以一行输出多段文本一样，也可以**一次输入多个值**：

```cpp
#include <iostream>

int main()
{
    std::cout << "Enter two numbers separated by a space: ";

    int x{};
    int y{};
    std::cin >> x >> y; // 读入两个数，分别存进 x 和 y

    std::cout << "You entered " << x << " and " << y << '\n';

    return 0;
}
```

输出示例：

```
Enter two numbers separated by a space: 5 6
You entered 5 and 6
```

输入的值之间用**空白字符**（空格、制表符、换行）分隔即可。

> **关键洞察**
>
> 关于“**在马上要用 `std::cin` 覆盖它之前，是否还有必要先初始化变量**”，社区里有些讨论。因为用户输入的值会直接覆盖初始值，看似初始化多余。但按照我们“**永远初始化变量**”的建议，最佳实践仍然是：**先初始化**。

> **进阶阅读**
>
> C++ 的 I/O 库**不支持在用户不按回车的情况下**接收键盘输入。如果你有这种需求，就得使用第三方库。对于控制台程序，我们推荐 pdcurses、FXTUI、cpp-terminal 或 notcurses。很多图形界面库自己也有类似的功能。

## std::cin 也是带缓冲的

前面提到过，**输出**其实是两个步骤：

1. 每次输出请求的数据会被**追加**到输出缓冲区的**末尾**；
2. 稍后，缓冲区**前端**的数据会被 flush 到输出设备（控制台）。

> **关键洞察**
>
> 从**末尾**加入、从**前端**取出，保证数据按加入的顺序被处理。这被称为 **FIFO（先进先出，first in, first out）**。

类似地，**输入**也是两个步骤：

1. 你按下的每个字符会被追加到**输入缓冲区（在 `std::cin` 内部）** 的末尾。你按下回车键提交输入时，这个回车也会以 `'\n'` 的形式存进缓冲区。
2. **提取运算符 `>>`** 从输入缓冲区的**前端**取字符，把它们**转换**成一个值，然后（通过拷贝赋值）赋给对应的变量。之后这个变量就可以被使用了。

> **关键洞察**
>
> 输入缓冲区中**每一行输入都以 `'\n'` 结尾**。

我们用下面这段程序来演示：

```cpp
#include <iostream>

int main()
{
    std::cout << "Enter two numbers: ";

    int x{};
    std::cin >> x;

    int y{};
    std::cin >> y;

    std::cout << "You entered " << x << " and " << y << '\n';

    return 0;
}
```

这个程序分**两条语句**分别读入两个变量。我们把它运行两次。

**运行 #1：** 执行到 `std::cin >> x;` 时，程序等待输入。输入 `4` 并回车。`4\n` 进入输入缓冲区，`4` 被提取到 `x`。

执行到 `std::cin >> y;` 时，程序再次等待输入。输入 `5` 并回车。`5\n` 进入输入缓冲区，`5` 被提取到 `y`。最后程序打印 `You entered 4 and 5`。

这一轮里没有意外。

**运行 #2：** 执行到 `std::cin >> x;` 时，程序等待输入。这次输入 `4 5` 并回车。`4 5\n` 进入输入缓冲区，但**只有 `4` 被提取给 `x`**（提取在空格处停下）。

执行到 `std::cin >> y;` 时，程序**不会再等待输入**。因为输入缓冲区里**还剩着 `5`**，它会被提取给 `y`。然后程序打印 `You entered 4 and 5`。

注意在第二次运行中，程序在给 `y` 提取时**没有再等用户输入**，因为缓冲区里已经有可用的数据了。

> **关键洞察**
>
> `std::cin` 采用缓冲，是为了**把“输入数据”与“提取数据”这两件事分开**：我们可以**一次输入多份数据**，然后进行**多次提取**。

## 基本的提取过程

下面是 `operator >>` 对输入的简化处理过程：

1. 如果 `std::cin` 处于**不良状态**（例如上次提取失败且尚未清除），则**不进行提取**，提取过程立即中止。
2. 从输入缓冲区**前端**丢弃开头的**空白字符**（空格、制表符、换行）。这会把上一行输入残留下来的换行符扔掉。
3. 如果这时缓冲区**空了**，`operator >>` 会等用户输入更多数据。用户新输入的数据前面的空白也会被丢弃。
4. `operator >>` 会**尽可能多地**连续提取字符，直到遇到换行符（表示行末）或者一个**对目标变量来说不合法的字符**。

提取的结果如下：

- 如果第 1 步就中止了，则**根本没有发生提取**，什么都不做；
- 如果第 4 步中**提取到了字符**，提取成功。这些字符被转换成一个值，然后拷贝赋值给变量；
- 如果第 4 步中**一个字符都提不出来**，提取**失败**。被提取的对象会被赋值为 `0`（C++11 起如此），**之后的所有提取都会立刻失败**，直到 `std::cin` 被清除。

**任何未被提取的字符（包括换行）**会保留在缓冲区里，供下一次提取使用。

> **相关内容**
>
> 关于如何**检测并处理提取失败、处理多余的输入、清除 `std::cin`**，我们会在 9.5 — std::cin 与非法输入处理 中讨论。

以下面这段代码为例：

```cpp
int x{};
std::cin >> x;
```

三种不同输入的结果：

- 用户输入 `5a` 并回车：缓冲区变成 `5a\n`。`5` 被提取、转换为整数、赋给 `x`。`a\n` 留在缓冲区里等下次提取。
- 用户输入 `b` 并回车：缓冲区变成 `b\n`。`b` 不是合法整数，一个字符都提取不出来，**提取失败**。`x` 被设为 `0`，且**之后的提取都会失败**，直到清空输入流。
- 如果 `std::cin` 因上一次提取失败而处于**不良状态**，那么这里**什么都不会发生**，`x` 的值也不会被改变。

后面小测验里我们会看到更多例子。

## operator<< 与 operator>>

初学者经常把 `std::cin`、`std::cout`、`<<`、`>>` 搞混。这里给一个简单的记忆方法：

- `std::cin` 和 `std::cout` 永远出现在运算符**左边**。
- `std::cout` 用于**输出**（cout = character output）。
- `std::cin` 用于**输入**（cin = character input）。
- `<<` 和 `std::cout` 一起使用，表示数据流动方向。`std::cout << 4` 表示把 `4` 送到控制台。
- `>>` 和 `std::cin` 一起使用，也表示数据流动方向。`std::cin >> x` 表示把用户从键盘输入的值送进变量 `x`。

关于运算符的更多内容，我们会在 1.9 — 字面量与运算符简介 中讲到。
