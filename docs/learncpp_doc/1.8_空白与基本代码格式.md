# 1.8 — 空白与基本代码格式

---

**空白（whitespace）** 是一个泛指、用于格式化目的的字符集合。在 C++ 中，它主要指**空格、制表符（Tab）和换行**。C++ 中的空白通常有三个作用：**分隔某些语言元素、出现在文本中、以及用于格式化代码**。

---

## 某些语言元素必须由空白分隔

C++ 的语法要求**某些元素之间必须有空白**，这大多发生在两个关键字或标识符要连着出现的地方——这样编译器才能把它们分开来。

例如，变量声明必须用空白分隔：

```cpp
int x; // int 和 x 必须用空白分隔
```

如果你写成 `intx`，编译器会把它当成一个**标识符**，然后抱怨自己不知道 `intx` 是啥。

再比如，函数的**返回类型**和**函数名**也必须用空白分隔：

```cpp
int main(); // int 和 main 必须用空白分隔
```

在**需要**空白作为分隔符时，编译器**不在乎你用了多少空白，只要有就行**。下面的变量定义**全部合法**：

```cpp
int x;
int                y;
            int
z;
```

在某些情况下，**换行**也会作为分隔符使用。例如**单行注释**是**以换行结束**的。看下面这个反例：

```cpp
std::cout << "Hello world!"; // This is part of the comment and
this is not part of the comment
```

第二行**并不属于注释**，会被当成代码去编译，从而出错。

**预处理器指令**（例如 `#include <iostream>`）必须**各自单独占一行**：

```cpp
#include <iostream>
#include <string>
```

---

## 引号内的文本会**原样保留**空白

在**引号包裹**的文本中，空白的数量会被**原样保留**。

```cpp
std::cout << "Hello world!";
```

与下面这一行**不同**：

```cpp
std::cout << "Hello          world!";
```

**换行不能出现在字符串字面量里**：

```cpp
std::cout << "Hello
     world!"; // 不允许！
```

如果两段被引号括起来的文本之间**只有空白（空格、制表符、换行）**，它们会被**自动拼接（concatenated）** 起来：

```cpp
std::cout << "Hello "
     "world!"; // 输出 "Hello world!"
```

---

## 用空白来格式化代码

除了上述场景外，**空白通常会被编译器忽略**。这意味着我们**可以随意使用空白**来格式化代码，让它更易读。

例如下面这段就非常难读：

```cpp
#include <iostream>
int main(){std::cout<<"Hello world";return 0;}
```

下面这样就好一点（但仍然很挤）：

```cpp
#include <iostream>
int main() {
std::cout << "Hello world";
return 0;
}
```

而下面这样就更清晰了：

```cpp
#include <iostream>

int main()
{
    std::cout << "Hello world";

    return 0;
}
```

如果需要，**一条语句也可以拆成多行**：

```cpp
#include <iostream>

int main()
{
    std::cout
        << "Hello world"; // 可以正常工作
    return 0;
}
```

这在**语句特别长**的时候很有用。

---

## 基本格式规范

与某些语言不同，C++ **不对程序员的代码格式做任何强制要求**。因此我们说 C++ 是一种 **“与空白无关（whitespace-independent）” 的语言**。

这既是好事也是坏事。好的一面是你想怎么写就怎么写；坏的一面是多年来出现了**很多种**格式化 C++ 代码的风格，而且大家对“哪种更好”常常**分歧巨大（甚至吵得头破血流）**。我们的基本原则是：**最好的风格就是能写出最易读、最一致代码的那种风格。**

下面是我们对基本格式的建议：

---

### 1）缩进：用 Tab 或空格都可以

- 缩进用 **Tab 或空格都行**（大多数 IDE 都有一个选项，可以把按下 Tab 键自动转换成若干空格）。
- **喜欢空格**的人认为空格能保证代码在**任何编辑器和设置下**都能精确对齐；
- **喜欢 Tab** 的人则会问：**用来做缩进的字符不就是 Tab 吗？** 而且你还可以把它的显示宽度设成任何你喜欢的值。
- **这里没有标准答案**——争论这个就跟争论蛋糕好吃还是派好吃差不多，最终纯属**个人偏好**。

不管你怎么选，我们建议**把 Tab 设置成相当于 4 个空格宽度的缩进**。有些 IDE 默认是 3 个空格，也完全可以。

---

### 2）函数大括号的两种常见风格

**第一种：左大括号 `{` 与语句同一行**：

```cpp
int main() {
    // 语句写在这里
}
```

这样做的理由是：**减少纵向空白**（不需要为一个左大括号单独占一整行），一屏就能容纳更多代码，也就少了滚动、便于理解。

**第二种（本教程采用）：左大括号 `{` 单独一行**：

```cpp
int main()
{
    // 语句写在这里
}
```

这种写法的好处是**可读性更好，也更不容易出错**——因为**成对的大括号会始终缩进在同一列**，一旦出现大括号不匹配的编译错误，一眼就能看出问题在哪。

---

### 3）大括号内的每条语句相对函数起始位置缩进一层

例如：

```cpp
int main()
{
    std::cout << "Hello world!\n";       // 缩进 1 个 Tab（4 个空格）
    std::cout << "Nice to meet you.\n";  // 缩进 1 个 Tab（4 个空格）
}
```

---

### 4）行不要太长

一直以来，**80 字符**都是行长的“事实标准”。如果一行代码要超过这个长度，应当**在合适的位置**拆成多行。拆行时，可以给**续行多加一层缩进**，或者当多行内容相似时，把续行**对齐到上一行**（哪种更易读就用哪种）：

```cpp
int main()
{
    std::cout << "This is a really, really, really, really, really, really, really, "
        "really long line\n"; // 续行多加一层缩进

    std::cout << "This is another really, really, really, really, really, really, really, "
                 "really long line\n"; // 续行与上一行文本对齐

    std::cout << "This one is short\n";
}
```

这样代码更易读。在现代的宽屏显示器上，也方便你**并排放两个窗口**，对比相似的代码。

> **最佳实践**
>
> **考虑把每一行的长度控制在 80 字符以内。**

> **小贴士**
>
> 很多编辑器都有一个功能（或者以插件/扩展的形式）叫做 **“列参考线（column guide）”**，可以在指定列（比如第 80 列）画一条竖线，让你一眼看出哪些行太长了。你可以搜索“你的编辑器名 + Column guide”看看你的编辑器怎么开启这个功能。

---

### 5）用运算符拆行时，把运算符放在**下一行的开头**

如果一行需要用运算符（例如 `<<` 或 `+`）拆开，应当把运算符放到**下一行的开头**，而不是当前行的末尾：

```cpp
std::cout << 3 + 4
    + 5 + 6
    * 7 * 8;
```

这样能让读者更清楚地看出这几行是**前一行的续行**，而且运算符会**在左侧对齐**，更方便阅读。

---

### 6）用空白让代码更易读——对齐值和注释、在代码块之间加空行

**较难读**：

```cpp
cost = 57;
pricePerItem = 24;
value = 5;
numberOfItems = 17;
```

**更易读**：

```cpp
cost          = 57;
pricePerItem  = 24;
value         = 5;
numberOfItems = 17;
```

**较难读**：

```cpp
std::cout << "Hello world!\n"; // cout lives in the iostream library
std::cout << "It is very nice to meet you!\n"; // these comments make the code hard to read
std::cout << "Yeah!\n"; // especially when lines are different lengths
```

**更易读**：

```cpp
std::cout << "Hello world!\n";                  // cout lives in the iostream library
std::cout << "It is very nice to meet you!\n";  // these comments are easier to read
std::cout << "Yeah!\n";                         // especially when all lined up
```

**较难读**：

```cpp
// cout lives in the iostream library
std::cout << "Hello world!\n";
// these comments make the code hard to read
std::cout << "It is very nice to meet you!\n";
// especially when all bunched together
std::cout << "Yeah!\n";
```

**更易读**：

```cpp
// cout lives in the iostream library
std::cout << "Hello world!\n";

// these comments are easier to read
std::cout << "It is very nice to meet you!\n";

// when separated by whitespace
std::cout << "Yeah!\n";
```

---

我们会在整个教程中**始终遵循这些约定**，你也会慢慢习惯它们。当我们讲到新的主题时，我们也会补充与之相关的新的风格建议。

归根结底，C++ **允许**你选择自己最舒服、或你觉得最好的风格。但我们**强烈建议**你使用与我们示例相同的风格——它已经被**成千上万的程序员**、**上亿行代码**反复验证过，是**为写好代码而优化**的选择。

**一个例外：** 如果你是在**别人的代码库**里工作，那就**用他们的风格**。**保持一致性比坚持你自己的偏好更重要**。

> **最佳实践**
>
> **在已有项目中，请与项目里已经采用的风格保持一致。**
